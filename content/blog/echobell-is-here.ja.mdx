# Echobell が登場！

Echobell をご紹介します。大事なものを見逃さないためのインスタントアラートアプリです。ビルド失敗、稼働監視のダウン、重要メール——見落としがちな出来事を、信頼できるタイムリーな通知に変えます。

## Echobell とは？

Echobell は、webhook やメールをトリガーに受け取り、必要な人に必要な通知を即時に届ける、プライバシー‑ファーストな通知アプリです。通知はチャンネルで整理し、テンプレートで内容を整え、通知の緊急度（通常・時間重視・通話）を選べます。実運用に耐える、素早く柔軟なワークフローに最適です。

要するに：チャンネルを作り、webhook かメールを送るだけ。Echobell がパーソナライズされたアプリ通知にして届けます。

## 主な機能

### 通知チャンネル

Echobell の中心はチャンネル。シンプルで強力なアラートのパイプラインです。各チャンネルは以下を備えます：

- 名前と色（すぐに識別）
- タイトル/本文テンプレート（{{server}}、{{message}}、{{timestamp}} などの変数に対応）
- webhook またはメールでのトリガー
- チームやフォロワーが購読できる共有用リンク

### 複数のトリガー方法

Echobell は 2 つの確実な方法で通知を起動できます：

**Webhook 通知**：固有の URL を GET/POST で呼び出すだけで、チャンネル購読者へ通知。用途：

- サーバー監視アラート
- CI/CD パイプライン通知
- IoT デバイスのステータス更新
- EC の注文通知

**メール → 通知**：チャンネルのアドレスにメールを送信/転送して通知を起動（コード不要）。用途：

- 重要メールの通知化
- コーディング不要の通知フロー構築
- メールは対応だが webhook 非対応のレガシー連携

### カスタマイズ可能なテンプレート

テンプレートと変数で、わかりやすい通知に：

```
タイトル：{{server}} ステータスアラート
本文：{{message}} を {{timestamp}} に検出
```

Webhook の body や query の変数、あるいはメールの subject/from/text を使って、明確でパーソナライズされた通知を生成できます。

### 柔軟な通知タイプ

アラートは 3 種類：

- **通常（Active）**：日常の更新に最適な標準通知
- **時間重視**：一部の集中モードを突破する高優先度通知
- **通話**：重大インシデント向けの電話のような通知

### プライバシー重視

プライバシーは設計に組み込み済み：

- サーバーに保存するのはアカウント・チャンネル・購読情報のみ
- 通知内容と履歴は端末上にのみ保存
- センシティブなデータは常にあなたの管理下

## ユースケース

Echobell は様々な現場のワークフローにフィットします：

### 開発者向け

- Webhook 通知でのサーバー/稼働監視（Grafana、Prometheus）
- CI/CD・GitHub Actions のビルド成功/失敗アラート
- 本番障害やログスパイクの検知

### トレーダー向け

- 株式/暗号資産のリアルタイム価格アラート
- 大きな変動のしきい値通知
- メール転送を使ったメンション/シグナルの捕捉

### チーム向け

- プロジェクト/顧客ごとのチャンネル
- 希望する緊急度（通常・時間重視・通話）で購読
- 必要なときは集中モード中でも重要通知を通す

### 個人利用

- サイトダウン通知や問い合わせフォームの通知
- 受信箱に張り付かず重要メールの通知を受け取る
- 自分にとって大切な事柄のリマインド

## はじめ方

数分で始められます：

1. [App Store](https://apps.apple.com/app/id6743597198) から Echobell をダウンロード
2. メールまたは「Apple でサインイン」でアカウント作成
3. 最初のチャンネルを作成し、タイトル/本文テンプレートを設定
4. webhook またはメールのトリガーを追加
5. テスト送信して、インスタント通知を受け取り開始
