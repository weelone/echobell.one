# Echobell 正式发布！

我们很高兴与你分享 Echobell —— 一款帮助你剔除噪音、不错过重要事件的即时提醒应用。如果你曾错过构建失败、宕机告警或重要邮件，Echobell 会把这些时刻变成清晰、及时、值得信赖的通知。

## 什么是 Echobell？

Echobell 是一款隐私优先的通知应用，通过 Webhook 或电子邮件接收触发，并将合适的提醒即时送达给合适的人。你可以用频道来组织提醒，用模板来塑造内容，并选择通知的紧急程度。它快速、灵活，贴合真实工作流。

一句话概括：创建频道，发送 Webhook 或邮件，Echobell 就会把它变成个性化的应用通知——普通、时间敏感，甚至来电式提醒。

## 核心功能

### 通知频道

Echobell 的核心是频道：简单而强大的提醒流水线。每个频道包括：

- 名称与颜色（便于快速识别）
- 标题与正文模板，支持 `{{server}}`、`{{message}}`、`{{timestamp}}` 等变量
- 通过 Webhook 或电子邮件触发
- 可分享的订阅链接，方便团队或关注者订阅

### 多种触发方式

Echobell 提供两种可靠的触发方式：

**Webhook 通知**：用 GET 或 POST 调用你的专属 URL，Echobell 会通知该频道的所有订阅者。适用于：

- 服务器监控告警
- CI/CD 流水线通知
- 物联网设备状态更新
- 电商订单提醒

**邮件 → 通知**：向频道邮箱发送/转发邮件即可触发提醒，无需写代码。适用于：

- 将重要邮件转化为通知
- 无需开发即可搭建通知流程
- 对接只支持邮件、不支持 Webhook 的遗留系统

### 可自定义的通知模板

借助模板与变量，让提醒既清晰又个性化：

```
标题：{{server}} 状态提醒
内容：{{message}} 于 {{timestamp}} 检测到
```

你可以在 Webhook 的请求体或查询字符串里传递变量，或使用邮件的 subject/from/text 字段来渲染通知内容。

### 灵活的通知类型

并非所有提醒都一样。Echobell 支持三种类型：

- **普通（Active）**：适用于日常更新的标准通知
- **时间敏感**：高优先级通知，可突破部分专注模式
- **来电式**：适用于关键事件的呼叫类提醒

### 隐私优先

隐私是内置的：

- 我们的服务器只保存账号、频道与订阅数据
- 通知内容与历史仅保存在你的设备上
- 敏感数据始终由你掌控

## 适用场景

Echobell 适合多种真实工作流：

### 面向开发者

- 借助 Webhook 通知进行服务器/可用性监控（Grafana、Prometheus）
- CI/CD 与 GitHub Actions 构建成功/失败提醒
- 生产环境错误与日志峰值告警

### 面向交易者

- 股票/加密货币价格的实时提醒
- 大幅波动的阈值通知
- 通过邮件转发捕捉提及与信号

### 面向团队

- 按项目或客户创建频道
- 根据偏好订阅通知紧急度（普通、时间敏感、来电式）
- 需要时，关键提醒可突破专注时间

### 个人使用

- 网站宕机与表单提交提醒
- 不必时刻看收件箱也能接收重要邮件通知
- 为任何对你重要的事项创建提醒
