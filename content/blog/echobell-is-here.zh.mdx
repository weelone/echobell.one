# Echobell 正式发布！

今天，我们很兴奋地宣布 Echobell 的正式发布 —— 这是一款革命性的即时提醒应用，旨在改变您接收和管理通知的方式。

## 什么是 Echobell？

Echobell 是一款为通知管理带来简便性和强大功能的即时提醒应用。在当今信息过载的世界中，跟踪真正重要的提醒确实具有挑战性。Echobell 通过提供一个灵活的平台来解决这个问题，该平台可以处理 webhook 或电子邮件作为触发器，然后向您和您的订阅者发送自定义通知。

Echobell 的核心理念非常简单：为不同类型的通知创建频道，自定义通知的显示方式，并选择您希望收到提醒的紧急程度。

## 主要功能

### 通知频道
Echobell 的核心是 **频道** —— 可定制的通知管道，您可以为不同的目的创建。每个频道都有：
- 独特的名称和颜色，便于识别
- 可定制的通知模板，决定发送哪些信息
- 通过 webhook 或电子邮件的灵活触发选项
- 可分享的订阅链接，其他人可以接收您的通知

### 多种触发方式
Echobell 提供了两种强大的通知触发方式：

**Webhooks**：通过使用我们的 webhook URL，几乎可以与任何系统集成。当这些 URL 被调用（通过简单的 GET 或 POST 请求），您的频道会向所有订阅者发送通知。这非常适合：
- 服务器监控提醒
- CI/CD 管道通知
- 物联网设备状态更新
- 电子商务订单通知

**电子邮件触发**：每个频道可以有一个专用的电子邮件地址。当发送电子邮件到此地址时，频道会被触发，并发送通知。这使得：
- 将重要的电子邮件转发为通知变得容易
- 无需编码即可创建通知流程
- 与仅支持电子邮件而不支持 webhook 的旧系统集成

### 可定制的通知模板
使用我们的模板系统和动态变量，使您的提醒更有意义：

```
标题：{{server}} 状态提醒
内容：{{message}} 检测到于 {{timestamp}}
```

当使用 `server=production` 和 `message=CPU usage at 95%` 这样的变量触发时，通知变得个性化且信息丰富。

### 灵活的通知类型
并非所有提醒都是一样的。Echobell 让订阅者可以选择三种通知类型：

- **普通**：常规提醒的标准应用通知
- **时间敏感**：高优先级通知，可以突破某些专注模式
- **呼叫**：类似电话的提醒，用于需要立即关注的紧急情况

### 隐私优先
我们在构建 Echobell 时将隐私放在首位：
- 只有用户账户、频道和订阅数据存储在我们的服务器上
- 通知内容和历史记录仅保存在您的设备上
- 您的敏感提醒数据永远不会离开您的控制

## 使用场景

Echobell 足够灵活，可以满足各种通知需求：

### 对于开发者
- 监控服务器健康状况，并在出现问题时接收即时提醒
- 在构建完成或测试失败时接收通知
- 在生产环境中发生关键错误时接收提醒

### 对于交易者
- 获取股票价格变动的实时提醒
- 监控加密货币价格，并在显著波动时接收通知
- 设置特定股票的新闻文章或社交媒体提及的提醒

### 对于团队
- 为不同的项目或系统创建频道
- 让团队成员根据他们的首选通知紧急程度进行订阅
- 确保即使在专注时间内也不会错过关键提醒

### 个人使用
- 在您的网站宕机时接收提醒
- 接收重要电子邮件的通知，而无需检查收件箱
- 为任何对您重要的事情创建自定义提醒

## 入门指南

开始使用 Echobell 非常简单：

1. 从 [App Store](https://apps.apple.com/app/id6743597198) 下载 Echobell
2. 使用电子邮件创建账户或使用 Apple 登录
3. 使用自定义模板创建您的第一个频道
4. 设置 webhook 或电子邮件触发器
5. 开始接收通知！
