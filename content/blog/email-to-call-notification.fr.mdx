# Transformez vos e‑mails importants en notifications d’appel avec Echobell

Il y a toujours ce message qui ne peut pas attendre. Echobell convertit ces e‑mails critiques en notifications de type appel qui font vraiment sonner votre téléphone, même en mode concentration. Parfait pour les incidents de production, les expéditeurs VIP, les échecs de paiement et les alertes d’astreinte pour les développeurs.

À la fin, vous aurez :

- Un canal Echobell déclenché par e‑mail
- Des modèles clairs et concis, lus à voix haute dans les alertes d’appel
- Des notifications d’appel (Calling) avec nouvelles tentatives optionnelles
- Des Conditions intelligentes pour filtrer par expéditeur, objet ou mots‑clés
- Des règles de transfert simples pour Gmail, iCloud Mail ou Outlook

Confidentialité d’abord : Echobell ne stocke que votre compte, vos canaux et vos abonnements. Le contenu et l’historique des notifications restent sur vos appareils.

## Prérequis

- Installez Echobell depuis l’App Store : https://apps.apple.com/app/id6743597198
- Autorisez les notifications

## Étape 1 — Créer un canal

1. Ouvrez Echobell → Nouveau canal
2. Donnez un nom et une couleur (par exemple « Alertes e‑mail critiques »)
3. Enregistrez

## Étape 2 — Lier le déclencheur e‑mail

Sur la page du canal, copiez l’adresse e‑mail dédiée. Tout e‑mail envoyé à cette adresse déclenche le canal instantanément.

Variables disponibles dans les modèles lors d’un déclenchement par e‑mail :

- `from` : adresse de l’expéditeur
- `to` : adresse liée au canal
- `subject` : objet
- `text` : corps en texte brut
- `html` : corps HTML (le cas échéant)

## Étape 3 — Rédiger des modèles clairs et adaptés à la lecture vocale

Utilisez les variables pour fournir le contexte sans alourdir – les alertes d’appel lisent votre contenu à voix haute.

Exemple de titre :

```
[Alerte e‑mail] {{subject}}
```

Exemple de corps :

```
De : {{from}}
Aperçu : {{text}}
```

Astuces :

- Besoin d’un lien cliquable dans les enregistrements ? Passez `externalLink` via webhook ou définissez un Modèle de lien dans les paramètres avancés.
- Préférez des textes courts ; les longs paragraphes sont plus difficiles à suivre à l’écoute.

## Étape 4 — Choisir le type de notification Calling

Les abonnés choisissent leur mode d’alerte. Pour ne rien manquer :

- Type de notification : Calling
- Réglages de l’app → Répéter le contenu vocal : Activé (optionnel)
- Réglages de l’app → Réessayer les appels échoués : Activé (recommandé)

Les notifications de type Calling sonnent comme un appel et peuvent percer les modes de concentration selon vos réglages iOS.

## Étape 5 — Filtrer avec des Conditions (optionnel, recommandé)

Utilisez des Conditions pour n’appeler que lorsque c’est nécessaire. Ce sont des expressions évaluées sur les variables du modèle.

Exemples :

- Un expéditeur précis uniquement :
  ```
  from == "alerts@yourcompany.com"
  ```
- Uniquement si l’objet contient « URGENT » :
  ```
  subject.toLowerCase().includes("urgent")
  ```
- Combinaison de règles :
  ```
  (from == "pager@service.com" || from == "vip@customer.com") && subject.toLowerCase().includes("fail")
  ```

Sans Conditions, chaque e‑mail envoyé à l’adresse du canal déclenche une notification.

## Étape 6 — Configurer le transfert d’e‑mails (choisissez votre fournisseur)

Transférez les e‑mails importants vers l’adresse du canal. Options courantes :

### Gmail

- Gmail → Paramètres → Voir tous les paramètres → Filtres et adresses bloquées
- Créez un filtre (De / Objet contient)
- Action : Transférer vers l’adresse e‑mail du canal Echobell
- Optionnel : Ajouter un libellé ou ignorer la boîte de réception

Remarque : Gmail peut exiger la vérification de l’adresse de transfert. Si ce n’est pas pratique, passez via un alias de groupe que vous contrôlez.

### iCloud Mail

- iCloud Mail → Réglages → Règles
- Si : De/Objet contient vos conditions
- Alors : Transférer vers l’adresse e‑mail du canal Echobell

### Outlook

- Outlook/Exchange → Paramètres → Courrier → Règles
- Ajoutez des conditions sur expéditeur/objet/mots‑clés
- Action : Transférer/Rediriger vers l’adresse du canal Echobell

## Étape 7 — Tester votre alerte

- Envoyez un e‑mail de test à l’adresse du canal
- Vous devriez recevoir une alerte de type appel lisant vos modèles
- Consultez l’enregistrement dans Echobell ; ajustez modèles/conditions si besoin

## Dépannage

- Pas d’appels :
  - Vérifiez que le type de notification de votre abonnement est Calling
  - Contrôlez les autorisations iOS et les modes de concentration
  - Activez « Réessayer les appels échoués » dans les réglages
- Le modèle ne s’affiche pas :
  - Vérifiez les variables : `from`, `to`, `subject`, `text`, `html`
  - Les Conditions sont des expressions simples — ne les entourez pas de `{{ }}`
- Trop bruyant :
  - Renforcez vos Conditions
  - Ajoutez des mots‑clés d’objet ou des expéditeurs précis dans vos règles

## Où c’est idéal

- Alertes d’astreinte et incidents de production
- Notifications de monitoring envoyées par e‑mail (et webhooks si besoin)
- Messages clients/VIP à ne pas rater

## Prochaines étapes

- Créez des canaux distincts par équipe ou par système
- Utilisez les webhooks pour les alertes système et l’e‑mail pour les messages humains
- Partagez le lien d’abonnement du canal avec les collègues qui ont besoin des appels
