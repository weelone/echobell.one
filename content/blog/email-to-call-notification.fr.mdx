# Transformer les e-mails importants en notifications d’appel

Vous ne voulez plus jamais manquer un e-mail urgent ? Avec Echobell, transformez vos e-mails importants en notifications de type appel qui sonnent comme un vrai appel. Parfait pour les incidents de production, les expéditeurs VIP, les échecs de paiement et toute alerte critique.

## Ce que vous allez configurer

- Un canal déclenché par e-mail
- Des modèles pour le titre et le corps lus à voix haute
- Des notifications d’appel qui percent les modes de concentration
- Des Conditions facultatives pour filtrer par expéditeur/sujet
- Des règles de transfert dans Gmail, iCloud Mail ou Outlook

## Prérequis

- Echobell installé : App Store https://apps.apple.com/app/id6743597198
- Autorisation de notifications accordée

## Étape 1 — Créer un canal

1. Ouvrez Echobell → Nouveau canal
2. Nommez-le et choisissez une couleur (ex. « Alertes e-mail critiques »)
3. Enregistrez

## Étape 2 — Lier le déclencheur e-mail

Sur la page de détail du canal, copiez l’adresse e-mail dédiée. Tout e-mail envoyé à cette adresse déclenche le canal.

Quand un e-mail déclenche un canal, les variables suivantes sont disponibles :

- `from` : adresse de l’expéditeur
- `to` : adresse liée au canal
- `subject` : objet
- `text` : corps en texte brut
- `html` : corps HTML (le cas échéant)

## Étape 3 — Écrire des modèles clairs

Utilisez les variables pour rendre l’alerte utile.

Exemple de titre :

```
[Alerte e-mail] {{subject}}
```

Exemple de corps :

```
De : {{from}}
Aperçu : {{text}}
```

Astuces :

- Pour ajouter un lien aux enregistrements, passez `externalLink` via webhook ou définissez un Modèle de lien dans les Paramètres avancés.
- Pour les e-mails, restez concis ; les alertes d’appel liront ce contenu.

## Étape 4 — Choisir le type de notification Appel

Les abonnés choisissent leur mode de notification. Pour ne rien manquer :

- Type de notification : Appel
- Réglages de l’app → Répéter le contenu vocal : Activé (optionnel)
- Réglages de l’app → Réessayer les appels échoués : Activé (recommandé)

Les notifications d’appel sonnent comme un appel et peuvent percer les modes de concentration selon la configuration système.

## Étape 5 — Filtrer avec des Conditions (optionnel, recommandé)

Utilisez des Conditions pour appeler seulement pour certains e-mails. Ce sont des expressions évaluées sur les variables de modèle.

Exemples :

- Seulement pour un expéditeur précis :
  ```
  from == "alerts@yourcompany.com"
  ```
- Seulement si l’objet contient « URGENT » :
  ```
  subject.toLowerCase().includes("urgent")
  ```
- Règles multiples :
  ```
  (from == "pager@service.com" || from == "vip@customer.com") && subject.toLowerCase().includes("fail")
  ```

Sans Conditions, tous les e-mails envoyés au canal déclenchent des notifications.

## Étape 6 — Configurer le transfert des e-mails

Transférez les e-mails importants vers l’adresse du canal. Fournisseurs courants :

### Gmail

- Gmail → Paramètres → Voir tous les paramètres → Filtres et adresses bloquées
- Créez un filtre (De / Objet contient)
- Action : Transférer vers l’adresse e-mail de votre canal Echobell
- Optionnel : Ajouter un libellé ou ignorer la boîte de réception

Remarque : Gmail peut exiger la vérification de l’adresse de transfert. Si ce n’est pas pratique, utilisez un alias de groupe que vous contrôlez.

### iCloud Mail

- iCloud Mail → Réglages → Règles
- Si : De/Objet contient…
- Alors : Transférer vers l’adresse e-mail du canal Echobell

### Outlook

- Outlook/Exchange → Paramètres → Courrier → Règles
- Ajoutez des conditions sur l’expéditeur/l’objet/mots-clés
- Action : Transférer/Rediriger vers l’adresse du canal Echobell

## Étape 7 — Tester

- Envoyez un e-mail de test à l’adresse du canal
- Vous devriez recevoir une alerte de type appel avec lecture des modèles
- Consultez l’enregistrement dans Echobell ; ajustez modèles/conditions si nécessaire

## Dépannage

- Pas d’appels :
  - Vérifiez que votre abonnement utilise le type Appel
  - Vérifiez les autorisations iOS et les modes de concentration
  - Activez « Réessayer les appels échoués » dans les Réglages
- Le modèle ne s’affiche pas :
  - Vérifiez les variables : `from`, `to`, `subject`, `text`, `html`
  - Validez les expressions dans Conditions (sans `{{ }}` dans Conditions)
- Trop bruyant :
  - Renforcez les Conditions
  - Ajoutez des mots-clés ou des expéditeurs spécifiques dans vos règles mail

## Confidentialité

Echobell ne stocke sur le serveur que votre compte, vos canaux et vos abonnements. Le contenu des notifications par e-mail et l’historique restent sur vos appareils.

## Prochaines étapes

- Partagez le lien d’abonnement du canal avec vos coéquipiers
