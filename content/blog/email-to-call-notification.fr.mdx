Certains emails doivent vous parvenir quoi qu'il arrive — incidents de production, échecs de facturation, expéditeurs VIP. Echobell vous permet de transférer ces emails vers un canal et de les recevoir comme des notifications d'appel qui sonnent même en mode Concentration.

Echobell ne stocke côté serveur que les comptes, les canaux et les abonnements. Le contenu des notifications reste sur vos appareils.

## Prérequis

- Echobell installé ([App Store](https://apps.apple.com/app/apple-store/id6743597198?pt=128151925&ct=blog-email-to-call-notification-fr&mt=8))
- Autorisation de notifications accordée

## Étape 1 — Créer un canal

1. Ouvrez Echobell → Nouveau canal
2. Nommez-le (ex. « Alertes Email Critiques ») et choisissez une couleur
3. Enregistrez

## Étape 2 — Lier le déclencheur email

Sur la page de détail du canal, copiez l'adresse email dédiée. Tout email envoyé à cette adresse déclenche le canal.

Variables de modèle disponibles pour les déclencheurs email :

- `from` — adresse de l'expéditeur
- `to` — adresse liée au canal
- `subject` — objet de l'email
- `text` — corps en texte brut
- `html` — corps HTML (si présent)

## Étape 3 — Rédiger des modèles adaptés à la voix

Les notifications d'appel lisent ce contenu à voix haute ; soyez concis et direct.

Titre :

```
[Email Alert] {{subject}}
```

Corps :

```
From: {{from}}
Preview: {{text}}
```

Pour associer un lien cliquable à l'enregistrement de notification, passez `externalLink` via Webhook ou définissez un modèle de lien dans les paramètres avancés.

## Étape 4 — Définir le type de notification Appel

Dans les paramètres de votre abonnement, configurez :

- Type de notification : **Appel**
- Paramètres app → Répéter le contenu vocal : Activé (optionnel)
- Paramètres app → Réessayer l'appel échoué : Activé (recommandé)

Les notifications d'appel sonnent comme un appel téléphonique et peuvent passer outre les modes Concentration selon les paramètres iOS.

## Étape 5 — Filtrer avec les Conditions (optionnel)

Sans Conditions, chaque email envoyé au canal déclenche une notification. Ajoutez une Condition pour restreindre les emails qui déclenchent réellement une sonnerie.

Expéditeur spécifique uniquement :

```
from == "alerts@yourcompany.com"
```

Objet contenant « URGENT » :

```
subject.toLowerCase().includes("urgent")
```

Règles combinées :

```
(from == "pager@service.com" || from == "vip@customer.com") && subject.toLowerCase().includes("fail")
```

## Étape 6 — Configurer le transfert d'email

### Gmail

1. Paramètres → Voir tous les paramètres → Filtres et adresses bloquées
2. Créez un filtre par expéditeur ou objet
3. Action : Transférer vers l'adresse email de votre canal Echobell
4. Optionnel : Ignorer la boîte de réception ou appliquer un libellé

Gmail peut exiger la vérification de l'adresse de transfert. Si c'est contraignant, transférez via un alias de groupe que vous contrôlez.

### iCloud Mail

1. Paramètres → Règles
2. Condition : L'expéditeur ou l'objet contient vos critères
3. Action : Transférer vers l'adresse email de votre canal Echobell

### Outlook

1. Paramètres → Courrier → Règles
2. Ajoutez des conditions sur l'expéditeur, l'objet ou des mots-clés
3. Action : Transférer/Rediriger vers l'adresse email de votre canal Echobell

## Étape 7 — Tester

1. Envoyez un email à l'adresse du canal (ou déclenchez votre règle de transfert)
2. Confirmez qu'une alerte sous forme d'appel arrive
3. Vérifiez l'enregistrement dans Echobell et ajustez les modèles ou conditions si nécessaire

## Dépannage

- **Pas d'appel reçu :** Vérifiez que le type de notification est défini sur Appel ; contrôlez les autorisations iOS et le mode Concentration ; activez Réessayer l'appel échoué dans les paramètres app.
- **Le modèle ne s'est pas rendu :** Les noms de variables sont `from`, `to`, `subject`, `text`, `html`. Les Conditions utilisent des expressions simples — pas de `{{ }}`.
- **Trop de bruit :** Resserrez les Conditions ou ajoutez des critères d'expéditeur/objet plus stricts dans vos règles de messagerie.

## Prochaines étapes

- Créez des canaux séparés par équipe ou système
- Utilisez les Webhooks pour les alertes système automatisées, l'email pour les messages humains
- Partagez le lien d'abonnement du canal avec les coéquipiers ayant besoin des mêmes alertes d'appel
