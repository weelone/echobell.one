# 重要なメールを電話通知に変換する

緊急メールを絶対に見逃したくない？Echobell なら、重要メールを“電話のように鳴る”通知に変換できます。プロダクション障害、VIP 送信者、決済失敗などの見逃せないアラートに最適です。

## セットアップ内容

- メールでトリガーされるチャンネル
- 読み上げ用のタイトル／本文テンプレート
- フォーカスモード中でも鳴る電話型通知
- 送信者／件名で絞り込むオプションの条件
- Gmail・iCloud Mail・Outlook の転送ルール

## 前提条件

- Echobell をインストール（App Store: https://apps.apple.com/app/id6743597198）
- 通知の許可をオン

## ステップ 1 — チャンネルを作成

1. Echobell → 新規チャンネル
2. 名前と色を設定（例：「重要メールアラート」）
3. 保存

## ステップ 2 — メールトリガーを紐づけ

チャンネル詳細ページで専用のメールアドレスをコピー。そこへ届いたメールがチャンネルをトリガーします。

メールでトリガーされた場合、テンプレートで以下の変数が使えます。

- `from`: 送信者アドレス
- `to`: チャンネルのアドレス
- `subject`: 件名
- `text`: プレーンテキスト本文
- `html`: HTML 本文（ある場合）

## ステップ 3 — わかりやすいテンプレート

変数を使って意味のあるアラートにしましょう。

タイトル例：

```
[Email Alert] {{subject}}
```

本文例：

```
From: {{from}}
Preview: {{text}}
```

Tips:

- レコードにリンクを出すなら、Webhook で `externalLink` を渡すか、詳細設定のリンクテンプレートを利用。
- 電話通知ではこの内容が読み上げられるため、簡潔に。

## ステップ 4 — 通知タイプ「電話」を選択

購読者は通知方法を選べます。見逃し防止には：

- 通知タイプ：電話
- アプリ設定 → 音声内容を繰り返す：オン（任意）
- アプリ設定 → 失敗した電話を再試行：オン（推奨）

電話型通知は電話のように鳴り、システム設定によってはフォーカスモードを突破できます。

## ステップ 5 — 条件でフィルタ（任意・推奨）

特定のメールだけを電話通知に。条件はテンプレート変数に対する式です。

例：

- 特定の送信者のみ：
  ```
  from == "alerts@yourcompany.com"
  ```
- 件名に「URGENT」を含むとき：
  ```
  subject.toLowerCase().includes("urgent")
  ```
- 複合条件：
  ```
  (from == "pager@service.com" || from == "vip@customer.com") && subject.toLowerCase().includes("fail")
  ```

条件が空なら、チャンネル宛てのすべてのメールで通知されます。

## ステップ 6 — メール転送を設定

重要メールをチャンネルのアドレスへ転送します。主なプロバイダ：

### Gmail

- Gmail → 設定 → すべての設定を表示 → フィルタとブロック中のアドレス
- フィルタを作成（From／件名に含む）
- アクション：Echobell チャンネルのアドレスへ転送
- 任意：ラベル付与や受信トレイをスキップ

注：Gmail は転送先の確認を求める場合があります。面倒な場合は、管理するグループエイリアス経由の転送を検討してください。

### iCloud Mail

- iCloud Mail → 設定 → ルール
- 条件：From／件名に…を含む
- 実行：Echobell チャンネルのアドレスへ転送

### Outlook

- Outlook/Exchange → 設定 → メール → ルール
- 送信者／件名／キーワードの条件を追加
- アクション：Echobell チャンネルのアドレスへ転送／リダイレクト

## ステップ 7 — テスト

- チャンネルのアドレスへテストメール送信
- テンプレートが読み上げられる電話型アラートを受信するはずです
- Echobell のレコードで確認し、必要に応じてテンプレート／条件を調整

## トラブルシューティング

- 電話が鳴らない：
  - 購読で通知タイプが「電話」になっているか
  - iOS の通知許可とフォーカスモードを確認
  - 「失敗した電話を再試行」をオン
- テンプレートが反映されない：
  - 変数名：`from`, `to`, `subject`, `text`, `html`
  - 条件の式を確認（条件では `{{ }}` は不要）
- 通知が多すぎる：
  - 条件を厳しく
  - メール側のルールで送信者／キーワードを追加

## プライバシー

Echobell のサーバーにはアカウント・チャンネル・購読のみを保存。通知内容と履歴はあなたのデバイスに保存されます。
