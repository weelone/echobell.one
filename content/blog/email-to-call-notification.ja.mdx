本番インシデント、請求エラー、VIP送信者からのメールは、何をしていても必ず届く必要があります。Echobellを使えば、それらのメールをチャンネルに転送し、集中モードを突き破って着信スタイルの通知として受け取ることができます。

EchobellはアカウントCHannelAnd購読情報のみをサーバーに保存します。通知の内容はデバイス上にのみ保持されます。

## 前提条件

- Echobellのインストール（[App Store](https://apps.apple.com/app/id6743597198)）
- 通知の許可設定が完了していること

## ステップ1 — チャンネルの作成

1. Echobellを開き → 新しいチャンネルを作成
2. 名前をつけ（例：「Critical Email Alerts」）てカラーを選択
3. 保存

## ステップ2 — メールトリガーの設定

チャンネル詳細ページで専用のメールアドレスをコピーします。このアドレスに送信されたメールがすべてチャンネルのトリガーになります。

メールトリガーで使用できるテンプレート変数:

- `from` — 送信者アドレス
- `to` — チャンネルにバインドされたアドレス
- `subject` — メールの件名
- `text` — プレーンテキストの本文
- `html` — HTMLの本文（存在する場合）

## ステップ3 — 音声向けテンプレートの作成

着信通知はこの内容を音声で読み上げるため、短く直接的に記述します。

タイトル:

```
[Email Alert] {{subject}}
```

本文:

```
From: {{from}}
Preview: {{text}}
```

通知レコードにクリック可能なリンクを付けるには、Webhookで`externalLink`を渡すか、詳細設定でリンクテンプレートを設定します。

## ステップ4 — 着信通知タイプの設定

購読設定で以下を構成します:

- 通知タイプ: **着信（Calling）**
- アプリ設定 → 音声内容を繰り返す: オン（任意）
- アプリ設定 → 着信失敗時に再試行: オン（推奨）

着信通知は電話のように鳴り、iOSの設定によっては集中モードを突き破って届きます。

## ステップ5 — 条件によるフィルタリング（任意）

条件を設定しない場合、チャンネルへのすべてのメールが通知をトリガーします。条件を追加することで、実際に着信を鳴らすメールを絞り込めます。

特定の送信者のみ:

```
from == "alerts@yourcompany.com"
```

件名に「URGENT」が含まれる場合:

```
subject.toLowerCase().includes("urgent")
```

複合ルール:

```
(from == "pager@service.com" || from == "vip@customer.com") && subject.toLowerCase().includes("fail")
```

## ステップ6 — メール転送ルールの設定

### Gmail

1. 設定 → すべての設定を確認 → フィルタとブロック中のアドレス
2. 差出人または件名でフィルタを作成
3. アクション: Echobellチャンネルのメールアドレスに転送
4. 任意: 受信トレイをスキップまたはラベルを適用

Gmailでは転送先アドレスの確認が必要な場合があります。手間な場合は、管理しているグループエイリアス経由で転送してください。

### iCloud メール

1. 設定 → ルール
2. 条件: 差出人または件名が条件に一致する
3. アクション: Echobellチャンネルのメールアドレスに転送

### Outlook

1. 設定 → メール → ルール
2. 送信者、件名、またはキーワードの条件を追加
3. アクション: Echobellチャンネルのメールアドレスに転送/リダイレクト

## ステップ7 — テスト

1. チャンネルアドレスにメールを送信（または転送ルールをトリガー）
2. 着信スタイルのアラートが届くことを確認
3. Echobellのレコードを確認し、テンプレートや条件を必要に応じて調整

## トラブルシューティング

- **着信が届かない:** 通知タイプが「着信（Calling）」に設定されているか確認。iOSの通知許可と集中モードを確認。アプリ設定で「着信失敗時に再試行」を有効にする。
- **テンプレートがレンダリングされない:** 変数名は`from`、`to`、`subject`、`text`、`html`です。条件は通常の式で記述し、`{{ }}`で囲まない。
- **通知が多すぎる:** 条件を厳しくするか、メールルールの送信者・件名の基準を絞り込む。

## 次のステップ

- チームやシステムごとに別々のチャンネルを作成する
