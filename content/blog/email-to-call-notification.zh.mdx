# 把重要邮件变成电话通知

不想再错过任何紧急邮件？用 Echobell，可以把重要邮件转换为“像电话一样响”的通知。非常适合生产事故、VIP 发件人、扣款失败等必须立即看到的消息。

## 本文将完成

- 一个通过邮箱触发的频道
- 语音朗读的标题与正文模板
- 即使在专注模式也能响起的电话通知
- 可选的条件过滤（按发件人/主题）
- 在 Gmail、iCloud Mail、Outlook 中设置转发规则

## 前置条件

- 安装 Echobell（App Store: https://apps.apple.com/app/id6743597198）
- 已授予通知权限

## 第一步 — 创建频道

1. 打开 Echobell → 新建频道
2. 设置名称与颜色（如「关键邮件提醒」）
3. 保存

## 第二步 — 绑定邮箱触发

在频道详情页复制专属邮箱地址。任何发送到该地址的邮件都会触发频道。

邮件触发时，模板可用变量：

- `from`: 发件人地址
- `to`: 频道绑定地址
- `subject`: 邮件主题
- `text`: 纯文本正文
- `html`: HTML 正文（若存在）

## 第三步 — 编写清晰模板

用变量让提醒更有信息量。

标题模板示例：

```
[邮件提醒] {{subject}}
```

正文模板示例：

```
来自：{{from}}
预览：{{text}}
```

Tips：

- 如需在记录中展示链接，可通过 Webhook 传入 `externalLink`，或在高级设置里配置链接模板。
- 邮件提醒内容会被电话通知朗读，建议简洁明了。

## 第四步 — 选择「电话」通知类型

订阅者可选择通知方式。为了确保不错过：

- 通知类型：电话
- 应用设置 → 重复语音内容：开启（可选）
- 应用设置 → 重试失败的电话：开启（推荐）

电话通知会像来电一样响铃，并可根据系统设置突破部分专注模式。

## 第五步 — 用条件过滤（可选，推荐）

用条件表达式筛选需要电话提醒的邮件。

示例：

- 仅特定发件人：
  ```
  from == "alerts@yourcompany.com"
  ```
- 仅当主题包含「URGENT」：
  ```
  subject.toLowerCase().includes("urgent")
  ```
- 多条件组合：
  ```
  (from == "pager@service.com" || from == "vip@customer.com") && subject.toLowerCase().includes("fail")
  ```

若不设置条件，任何发到该频道地址的邮件都会触发通知。

## 第六步 — 设置邮件转发

将重要邮件转发到频道地址。常见邮箱：

### Gmail

- Gmail → 设置 → 查看所有设置 → 过滤器和已封锁地址
- 新建过滤器（发件人/主题包含）
- 动作：转发到你的 Echobell 频道地址
- 可选：添加标签或跳过收件箱

注意：Gmail 可能要求验证转发地址。如不方便，可改用你控制的群组别名转发。

### iCloud Mail

- iCloud Mail → 设置 → 规则
- 若：发件人/主题包含…
- 则：转发到 Echobell 频道邮箱

### Outlook

- Outlook/Exchange → 设置 → 邮件 → 规则
- 添加发件人/主题/关键词条件
- 动作：转发/重定向到 Echobell 频道邮箱

## 第七步 — 测试

- 向频道邮箱发送一封测试邮件
- 你会收到电话式提醒，并朗读模板内容
- 在 Echobell 中查看记录，按需调整模板/条件

## 故障排查

- 没有电话响起：
  - 确认订阅选择了「电话」通知类型
  - 检查 iOS 通知权限与专注模式
  - 打开「重试失败的电话」
- 模板未渲染：
  - 检查变量：`from`、`to`、`subject`、`text`、`html`
  - 校验条件表达式（条件中不需要 `{{ }}`）
- 通知太多：
  - 收紧条件
  - 在邮箱规则中增加特定发件人/关键字

## 隐私

Echobell 服务器仅保存账号、频道与订阅。邮件触发的通知内容和记录仅保存在你的设备上。
