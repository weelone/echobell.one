# 把重要邮件变成 Echobell 的电话式通知

总有些邮件，必须第一时间知道。Echobell 能把这些关键邮件转换成像电话一样响起的 Calling 通知——即使处于专注模式，也能把你“叫醒”。非常适合生产事故、VIP 客户来信、扣款失败，以及开发者值班告警。

读完本文你将获得：

- 一个由邮件触发的 Echobell 频道
- 适合语音朗读、简洁明了的通知模板
- Calling（电话式）通知，支持可选重试
- 可按发件人、主题、关键词过滤的智能条件（Conditions）
- Gmail、iCloud Mail、Outlook 的简易转发规则

先说隐私：Echobell 一直坚持隐私优先。服务器只保存账户、频道和订阅；通知内容与记录仅保存在你的设备上。

## 前置条件

- 从 App Store 安装 Echobell：https://apps.apple.com/app/id6743597198
- 授予通知权限

## 第一步 — 创建频道

1. 打开 Echobell → 新建频道
2. 设置名称与颜色（例如「关键邮件提醒」）
3. 保存

## 第二步 — 绑定邮箱触发

在频道详情页复制专属邮箱地址。发往该地址的邮件会立即触发频道。

邮件触发时可用的模板变量：

- `from`：发件人地址
- `to`：频道绑定地址
- `subject`：邮件主题
- `text`：纯文本正文
- `html`：HTML 正文（若存在）

## 第三步 — 写出好听、好懂的模板

用变量补充关键信息，同时保持简洁——电话通知会把内容读出来。

标题示例：

```
[邮件提醒] {{subject}}
```

正文示例：

```
来自：{{from}}
预览：{{text}}
```

Tips：

- 想在记录里显示可点击链接？可通过 Webhook 传入 `externalLink`，或在高级设置里配置链接模板。
- 长段落不利于收听，尽量短句、要点化。

## 第四步 — 选择 Calling（电话式）通知

订阅者可以选择通知方式。为了万无一失：

- 通知类型：Calling
- 应用设置 → 重复语音内容：开启（可选）
- 应用设置 → 重试失败的电话：开启（推荐）

Calling 通知会像来电一样响铃，并可根据 iOS 设置突破专注模式。

## 第五步 — 用条件过滤（可选，推荐）

只在重要时刻打电话。条件是对模板变量进行求值的表达式。

示例：

- 仅特定发件人：
  ```
  from == "alerts@yourcompany.com"
  ```
- 仅当主题包含「URGENT」：
  ```
  subject.toLowerCase().includes("urgent")
  ```
- 组合规则：
  ```
  (from == "pager@service.com" || from == "vip@customer.com") && subject.toLowerCase().includes("fail")
  ```

若不设置条件，发送到频道地址的所有邮件都会触发通知。

## 第六步 — 设置邮件转发（按提供商）

把重要邮件转发到频道地址。常见做法如下：

### Gmail

- Gmail → 设置 → 查看所有设置 → 过滤器和已封锁地址
- 新建过滤器（发件人 / 主题包含）
- 动作：转发到你的 Echobell 频道地址
- 可选：打标签或跳过收件箱

注意：Gmail 可能要求验证转发地址。如果不方便，可使用你管理的群组别名做转发。

### iCloud Mail

- iCloud Mail → 设置 → 规则
- 若：发件人 / 主题包含你的条件
- 则：转发到 Echobell 频道邮箱

### Outlook

- Outlook/Exchange → 设置 → 邮件 → 规则
- 添加发件人 / 主题 / 关键词条件
- 动作：转发 / 重定向到 Echobell 频道邮箱

## 第七步 — 测试你的提醒

- 向频道邮箱发送一封测试邮件
- 你会收到电话式提醒，朗读你配置的模板

- 在 Echobell 中查看记录；按需微调模板或条件

## 故障排查

- 没有电话响起：
  - 确认订阅的通知类型是 Calling
  - 检查 iOS 通知权限与专注模式
  - 开启「重试失败的电话」
- 模板未渲染：
  - 校验变量：`from`、`to`、`subject`、`text`、`html`
  - 条件是普通表达式，不要包 `{{ }}`
- 通知太多：
  - 收紧条件
  - 在邮箱规则里增加主题关键词或特定发件人

## 适用场景

- 开发者值班（On‑call）与生产事故告警
- 各类通过邮件发送的监控通知（需要的话可叠加 Webhook）
- 绝对不能错过的客户／VIP 消息
