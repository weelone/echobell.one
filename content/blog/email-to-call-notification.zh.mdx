有些邮件无论你在做什么都必须及时看到——生产故障、账单失败、VIP 发件人。Echobell 可以让你将这些邮件转发到频道，以来电式通知的形式响铃，即使在专注模式下也能穿透。

Echobell 仅在服务端存储账户、频道和订阅信息。通知内容保留在你的设备上。

## 前提条件

- 已安装 Echobell（[App Store](https://apps.apple.com/app/apple-store/id6743597198?pt=128151925&ct=blog-email-to-call-notification-zh&mt=8)）
- 已授权通知权限

## 第一步——创建频道

1. 打开 Echobell → 新建频道
2. 命名（如"紧急邮件告警"），选择颜色
3. 保存

## 第二步——绑定邮件触发

在频道详情页复制专用邮件地址。发送到该地址的任何邮件都会触发频道。

邮件触发可用的模板变量：

- `from` — 发件人地址
- `to` — 频道绑定地址
- `subject` — 邮件主题
- `text` — 纯文本正文
- `html` — HTML 正文（如有）

## 第三步——编写适合语音播报的模板

来电通知会朗读内容，因此要保持简短直接。

标题：

```
[Email Alert] {{subject}}
```

正文：

```
From: {{from}}
Preview: {{text}}
```

如需在通知记录中附加可点击的链接，可通过 Webhook 传入 `externalLink`，或在高级设置中设置链接模板。

## 第四步——设置来电通知类型

在订阅设置中配置：

- 通知类型：**来电**
- 应用设置 → 重复播报语音内容：开启（可选）
- 应用设置 → 重试失败来电：开启（推荐）

来电通知会像电话一样响铃，根据 iOS 设置可穿透专注模式。

## 第五步——设置条件过滤（可选）

不设置条件时，每封发到频道的邮件都会触发通知。添加条件可限制哪些邮件才真正响铃。

仅限特定发件人：

```
from == "alerts@yourcompany.com"
```

主题包含"URGENT"：

```
subject.toLowerCase().includes("urgent")
```

组合规则：

```
(from == "pager@service.com" || from == "vip@customer.com") && subject.toLowerCase().includes("fail")
```

## 第六步——配置邮件转发

### Gmail

1. 设置 → 查看所有设置 → 过滤器和屏蔽的地址
2. 按发件人或主题创建过滤器
3. 操作：转发到你的 Echobell 频道邮件地址
4. 可选：跳过收件箱或添加标签

Gmail 可能需要验证转发地址。如果操作不便，可通过你管理的群组别名进行转发。

### iCloud 邮件

1. 设置 → 规则
2. 条件：发件人或主题包含你的筛选条件
3. 操作：转发到你的 Echobell 频道邮件地址

### Outlook

1. 设置 → 邮件 → 规则
2. 添加发件人、主题或关键词条件
3. 操作：转发/重定向到你的 Echobell 频道邮件地址

## 第七步——测试

1. 发送邮件到频道地址（或触发转发规则）
2. 确认收到来电式通知
3. 查看 Echobell 中的记录，按需调整模板或条件

## 故障排查

- **未收到来电：** 确认通知类型已设置为"来电"；检查 iOS 通知权限和专注模式；在应用设置中开启"重试失败来电"。
- **模板未渲染：** 变量名为 `from`、`to`、`subject`、`text`、`html`。条件使用纯表达式——不需要 `{{ }}` 包裹。
- **通知过多：** 收紧条件，或在邮件规则中添加更严格的发件人/主题筛选。

## 下一步
