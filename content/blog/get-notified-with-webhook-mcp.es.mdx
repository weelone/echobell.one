# Recibe notificaciones cuando las tareas de IA finalicen con WebhookMCP y Echobell

Las tareas de IA largas — generación de código, análisis de datasets, refactorizaciones grandes — no necesitan supervisión constante. [WebhookMCP](https://github.com/noobnooc/webhook-mcp) es un servidor MCP que permite a los asistentes de IA activar notificaciones via webhook cuando el trabajo termina. Combinado con Echobell, recibes una notificación push o una llamada telefónica en el momento en que tu tarea se completa.

## Configuración

### 1. Obtén tu URL de Webhook de Echobell

1. Abre [Echobell](https://apps.apple.com/app/apple-store/id6743597198?pt=128151925&ct=blog-get-notified-with-webhook-mcp-es&mt=8) y crea un canal (p. ej., "AI Tasks")
2. Copia la URL del webhook desde la configuración del canal

### 2. Configura tu entorno de desarrollo

**VS Code** — agrega a `settings.json`:

```json
{
  "mcp": {
    "servers": {
      "notification": {
        "command": "npx",
        "args": ["-y", "webhook-mcp"],
        "env": {
          "WEBHOOK_URL": "your-echobell-webhook-url"
        }
      }
    }
  }
}
```

**Claude, Cursor o Windsurf** — agrega a tu configuración MCP:

```json
{
  "mcpServers": {
    "notification": {
      "command": "npx",
      "args": ["-y", "webhook-mcp"],
      "env": {
        "WEBHOOK_URL": "your-echobell-webhook-url"
      }
    }
  }
}
```

WebhookMCP se instala automáticamente via `npx` la primera vez que se usa.

### 3. Solicita notificaciones en tus prompts

Una vez configurado, incluye solicitudes de notificación de forma natural en tus prompts:

- "Envíame una notificación cuando este análisis esté completo"
- "Avísame cuando termines de generar el código"
- "Notifícame cuando hayas terminado con esta tarea"

WebhookMCP reconoce estas solicitudes y activa Echobell automáticamente.

## Tipos de notificación

Configura tu canal de Echobell según la urgencia de la tarea:

- **Push normal** — generación de código rutinaria, borradores de documentación
- **Urgente (Time-sensitive)** — tareas de análisis en las que estás esperando activamente
- **Llamada (Calling)** — tareas críticas donde necesitas atención inmediata sin importar el modo No Molestar

Para equipos, crea un canal compartido de Echobell y comparte el enlace de suscripción — todos reciben notificaciones cuando una tarea compartida se completa, con sus propias preferencias de notificación.

## Instalación rápida en VS Code

Para VS Code, visita el [repositorio de WebhookMCP en GitHub](https://github.com/noobnooc/webhook-mcp) y usa el botón "Install in VS Code". El asistente te pedirá tu URL de webhook de Echobell durante la configuración.

Para integraciones relacionadas, consulta [notificaciones de GitHub Actions](/en/blog/github-actions-notifications) y la [documentación de webhook](/en/docs/webhook).
