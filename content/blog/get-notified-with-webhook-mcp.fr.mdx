# Recevez des notifications quand vos tâches IA se terminent avec WebhookMCP et Echobell

Les tâches IA longues — génération de code, analyse de jeux de données, refactorisations importantes — ne nécessitent pas une surveillance constante. [WebhookMCP](https://github.com/noobnooc/webhook-mcp) est un serveur MCP qui permet aux assistants IA de déclencher des notifications Webhook lorsque le travail est terminé. Associé à Echobell, vous recevez une notification push ou un appel téléphonique dès que votre tâche s'achève.

## Configuration

### 1. Obtenez votre URL Webhook Echobell

1. Ouvrez [Echobell](https://apps.apple.com/app/id6743597198) et créez un canal (ex. « Tâches IA »)
2. Copiez l'URL Webhook depuis les paramètres du canal

### 2. Configurez votre environnement de développement

**VS Code** — ajoutez dans `settings.json` :

```json
{
  "mcp": {
    "servers": {
      "notification": {
        "command": "npx",
        "args": ["-y", "webhook-mcp"],
        "env": {
          "WEBHOOK_URL": "your-echobell-webhook-url"
        }
      }
    }
  }
}
```

**Claude, Cursor ou Windsurf** — ajoutez dans votre configuration MCP :

```json
{
  "mcpServers": {
    "notification": {
      "command": "npx",
      "args": ["-y", "webhook-mcp"],
      "env": {
        "WEBHOOK_URL": "your-echobell-webhook-url"
      }
    }
  }
}
```

WebhookMCP s'installe automatiquement via `npx` lors de la première utilisation.

### 3. Demandez des notifications dans vos prompts

Une fois configuré, incluez des demandes de notification naturellement dans vos prompts :

- « Envoie-moi une notification quand cette analyse est terminée »
- « Alerte-moi une fois que tu as fini de générer le code »
- « Notifie-moi quand tu as terminé cette tâche »

WebhookMCP reconnaît ces demandes et déclenche Echobell automatiquement.

## Types de notifications

Configurez votre canal Echobell en fonction de l'urgence des tâches :

- **Push normal** — génération de code de routine, rédaction de documentation
- **Urgent** — tâches d'analyse que vous attendez activement
- **Appel** — tâches critiques nécessitant une attention immédiate, même en mode Ne pas déranger

Pour les équipes, créez un canal Echobell partagé et partagez le lien d'abonnement — tout le monde est notifié quand une tâche partagée se termine, avec ses propres préférences de notification.

## Installation rapide pour VS Code

Pour VS Code, rendez-vous sur le [dépôt GitHub WebhookMCP](https://github.com/noobnooc/webhook-mcp) et utilisez le bouton « Install in VS Code ». L'assistant vous demande votre URL Webhook Echobell lors de la configuration.

Pour les intégrations connexes, voir [les notifications GitHub Actions](/en/blog/github-actions-notifications) et la [documentation Webhook](/en/docs/webhook).
