# 用 WebhookMCP 和 Echobell 在 AI 任务完成时收到通知

代码生成、数据集分析、大规模重构——这些耗时的 AI 任务不需要你全程盯着。[WebhookMCP](https://github.com/noobnooc/webhook-mcp) 是一个 MCP 服务器，可以让 AI 助手在任务完成时触发 Webhook 通知。搭配 Echobell，你可以在任务刚结束时就收到推送通知或来电提醒。

## 配置步骤

### 1. 获取 Echobell Webhook URL

1. 打开 [Echobell](https://apps.apple.com/app/apple-store/id6743597198?pt=128151925&ct=blog-get-notified-with-webhook-mcp-zh&mt=8)，创建一个频道（如"AI 任务"）
2. 从频道设置中复制 Webhook URL

### 2. 配置开发环境

**VS Code** — 添加到 `settings.json`：

```json
{
  "mcp": {
    "servers": {
      "notification": {
        "command": "npx",
        "args": ["-y", "webhook-mcp"],
        "env": {
          "WEBHOOK_URL": "your-echobell-webhook-url"
        }
      }
    }
  }
}
```

**Claude、Cursor 或 Windsurf** — 添加到 MCP 配置文件：

```json
{
  "mcpServers": {
    "notification": {
      "command": "npx",
      "args": ["-y", "webhook-mcp"],
      "env": {
        "WEBHOOK_URL": "your-echobell-webhook-url"
      }
    }
  }
}
```

WebhookMCP 首次使用时会通过 `npx` 自动安装。

### 3. 在提示词中请求通知

配置完成后，在提示词中自然地加入通知请求即可：

- "这个分析完成后给我发通知"
- "代码生成好了提醒我一下"
- "这个任务做完了通知我"

WebhookMCP 会识别这些请求，并自动触发 Echobell 通知。

## 通知类型

根据任务紧急程度配置 Echobell 频道：

- **普通推送** — 日常代码生成、文档草稿
- **时间敏感** — 你正在等待结果的分析任务
- **来电** — 关键任务，无论勿扰模式都需要立即处理

对于团队，创建一个共享 Echobell 频道并分享订阅链接——当共享任务完成时，所有人都会收到通知，并可设置各自的通知偏好。

## VS Code 快速安装

对于 VS Code，访问 [WebhookMCP GitHub 仓库](https://github.com/noobnooc/webhook-mcp)，点击"Install in VS Code"按钮。安装向导会在设置过程中提示你输入 Echobell Webhook URL。

相关集成请参考：[GitHub Actions 通知](/en/blog/github-actions-notifications) 和 [Webhook 文档](/en/docs/webhook)。
