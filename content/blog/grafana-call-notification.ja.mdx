# 重要なアラートを見逃さない：Grafana の Webhook を通じた電話通知の設定

重要なシステムを監視する際、メール通知だけでは不十分な場合があります。重要なアラートをより即座に、より注目を集める方法で通知する必要があります。そこで Echobell の電話通知が役立ちます。

## なぜ電話通知なのか？

- **即座の注目**: メールやプッシュ通知は簡単に見逃されがちですが、電話は無視するのが難しいです
- **重要アラートの認識**: 即座の対応が必要なシステムの重大な障害に最適です
- **24 時間 365 日のアラート**: 睡眠中でも重要なアラートを見逃すことがありません
- **チームエスカレーション**: 主要な担当者が応答しない場合、複数のチームメンバーへの呼び出しカスケードを設定できます

## Grafana 電話通知の設定

### 前提条件

1. アクティブなサブスクリプションを持つ Echobell アカウント
2. 管理者権限を持つ Grafana インスタンスへのアクセス
3. Grafana で設定済みのアラートルール

### 設定手順

1. **Webhook コンタクトポイントの作成**

   - Grafana で、Alerting → Contact points に移動します
   - "New contact point"をクリックします
   - コンタクトポイントのタイプとして"Webhook"を選択します
   - "Echobell 電話通知"と名前を付けます

2. **Webhook の設定**

   - URL を設定：`https://hook.echobell.one/t/xxxx`（あなたの Echobell チャンネルの webhook）

3. **統合のテスト**

   - Grafana の"Test"ボタンを使用してテスト通知を送信します
   - 数秒以内に電話が着信するはずです

4. **通知ポリシーの作成**
   - Alerting → Notification policies に移動します
   - 新しいポリシーを作成するか、既存のものを変更します
   - 希望するアラートルールに Echobell コンタクトポイントを追加します

## ベストプラクティス

- **アラートしきい値を慎重に設定**: 電話通知は真に重要なアラートのために予約すべきです
- **通話タイムアウトを設定**: アラート疲れを防ぐため、Echobell で適切なタイムアウト期間を設定します
- **連絡先リストを維持**: 緊急連絡先リストを最新の状態に保ちます
- **対応手順を文書化**: チームメンバーが電話を受けた際の対応手順を明確にします

## 高度な設定

以下の方法で電話通知をカスタマイズできます：

- チームメンバーとチャンネルを共有
- 応答のない通話の再試行を設定
- 異なるタイプのアラートに対して音声メッセージをカスタマイズ
- 静寂時間とバックアップ連絡先を設定

詳細な設定オプションについては、[開発者向けドキュメント](/ja/docs/developer/grafana)をご覧ください。

## 結論

電話通知は、重要なシステムのアラートに不可欠な層を追加します。Echobell と Grafana の統合により、重要なアラートが見逃されることなく、チームがインシデントに迅速に対応できます。

インシデント対応時間とシステムの信頼性を向上させるために、今すぐ電話通知の使用を開始しましょう。
