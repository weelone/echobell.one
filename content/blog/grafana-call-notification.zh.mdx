# 为 Grafana 警报启用电话通知

在监控关键系统时，有时电子邮件通知是不够的。你需要一种更即时、更引人注意的方式来接收关键警报。这就是 Echobell 电话通知发挥作用的地方。

## 为什么选择电话通知？

- **即时关注**：与容易被忽略的电子邮件或推送通知不同，电话很难被忽视
- **关键警报意识**：非常适合需要立即采取行动的关键系统故障
- **全天候警报**：确保你永远不会错过关键警报，即使在睡眠时间
- **团队升级**：如果主要联系人没有响应，可以设置多个团队成员的呼叫级联

## 设置 Grafana 电话通知

### 前提条件

1. 一个有效订阅的 Echobell 账户
2. 具有管理员权限的 Grafana 实例访问权限
3. Grafana 中已配置的警报规则

### 配置步骤

1. **创建 Webhook 联系点**

   - 在 Grafana 中，导航到 Alerting → Contact points
   - 点击 "New contact point"
   - 选择 "Webhook" 作为联系点类型
   - 将其命名为 "Echobell 电话通知"

2. **配置 Webhook**

   - 将 URL 设置为：`https://api.echobell.one/v1/grafana/call`
   - 添加你的 Echobell API 密钥作为自定义标头：
     ```
     Header: X-API-Key
     Value: 你的-echobell-api-密钥
     ```

3. **测试集成**

   - 使用 Grafana 中的 "Test" 按钮发送测试通知
   - 你应该在几秒钟内收到电话

4. **创建通知策略**
   - 转到 Alerting → Notification policies
   - 创建新策略或修改现有策略
   - 将 Echobell 联系点添加到你所需的警报规则中

## 最佳实践

- **谨慎设置警报阈值**：电话通知应该保留给真正的关键警报
- **配置呼叫超时**：在 Echobell 中设置适当的超时期限，以防止警报疲劳
- **维护联系人列表**：保持紧急联系人列表的最新状态
- **记录响应程序**：确保团队成员知道在收到电话时该怎么做

## 高级配置

你可以通过以下方式自定义电话通知：

- 设置团队升级的呼叫级联
- 配置未接来电的重试次数
- 为不同类型的警报自定义语音消息
- 设置安静时间和备用联系人

有关详细的配置选项，请访问我们的[开发者文档](/docs/developer/grafana)。

## 结论

电话通知为关键系统添加了重要的警报层。通过 Echobell 与 Grafana 的集成，你可以确保不会错过关键警报，并且你的团队可以快速响应事件。

立即开始使用电话通知，提高你的事件响应时间和系统可靠性。
