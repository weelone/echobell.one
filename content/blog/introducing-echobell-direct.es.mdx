[Echobell Direct](/en/docs/direct) es un modo de alerta personal que omite la configuración de canales por completo. Creas una clave Direct, obtienes una URL de webhook y envías un `title` + `body` via POST para recibir una notificación instantánea en tu dispositivo. Sin suscriptores, sin plantillas, sin configuración de enrutamiento.

Esta guía explica qué es Direct, cómo configurarlo y cuándo usarlo en lugar de los Canales.

## ¿Qué es Echobell Direct?

Cada clave Direct genera una URL de webhook única (`/d/{token}`). Envía una solicitud a esa URL y la notificación llega a tu dispositivo de inmediato.

Usa Direct cuando necesites:

- Notificaciones personales desde scripts, cron jobs o pipelines de CI
- Título y cuerpo personalizados por solicitud en lugar de plantillas fijas
- Configuración rápida sin tener que crear un canal completo

Si necesitas notificaciones compartidas, plantillas reutilizables o entrega condicional, usa [Canales via Webhooks](/en/docs/webhook) en su lugar.

## Configuración en 3 pasos

### Paso 1: Crea una clave Direct

Abre Echobell, toca **Direct** en la lista de canales y crea una clave. Nómbrala según el servicio al que pertenece (p. ej. `CI Server`, `Cron`, `AI Tasks`).

### Paso 2: Copia tu URL de webhook

Cada clave tiene una URL con el formato:

```text
https://hook.echobell.one/d/YOUR_KEY_TOKEN
```

Trata esta URL como un secreto — cualquier persona que la tenga puede enviarte notificaciones.

### Paso 3: Envía tu primera notificación

```bash
curl -X POST https://hook.echobell.one/d/YOUR_KEY_TOKEN \
  -H "Content-Type: application/json" \
  -d '{
    "title": "Deploy failed",
    "body": "api-service failed health check",
    "notificationType": "time-sensitive",
    "externalLink": "https://dashboard.example.com/deploys/123"
  }'
```

Referencia completa de campos: [API de Direct](/en/docs/direct)

## Niveles de urgencia

Direct admite `notificationType` para que puedas adaptar la urgencia al evento:

- `active` — actualizaciones regulares
- `time-sensitive` — alertas importantes que atraviesan el modo de concentración (Focus)
- `calling` — incidentes críticos (requiere suscripción premium activa; si no, cae a time-sensitive)

Consulta [Tipos de notificación](/en/docs/notification) para ver el comportamiento completo.

## Agregar contexto a las alertas

Incluye `externalLink` en el cuerpo de la solicitud para adjuntar una URL a la notificación — útil para enlazar directamente a un log de deploy, un dashboard o un pull request.

Para agentes de IA y tareas asíncronas de larga duración, Direct es una forma directa de recibir notificaciones cuando el trabajo termina, en lugar de hacer polling manualmente. Consulta también: [WebhookMCP + Echobell](/en/blog/get-notified-with-webhook-mcp).

## Direct vs Canales

| | Direct | Canales |
|---|---|---|
| Audiencia | Solo personal | Compartido con compañeros |
| Plantillas | Contenido por solicitud | Plantillas reutilizables y fijas |
| Tiempo de configuración | Menos de un minuto | Requiere configurar el canal |
| Condiciones | No admitido | Admitido |

Usa Canales cuando las alertas deban llegar a varias personas o cuando quieras formato consistente en un servicio. [Documentación de plantillas](/en/docs/template) · [Qué es Echobell](/en/docs/what-is-echobell)

## Prácticas de seguridad

- Almacena los tokens de Direct en variables de entorno o un gestor de secretos, no en el código
- Crea una clave por servicio para poder revocarlas individualmente
- Rota los tokens inmediatamente si una URL puede haberse filtrado
- Reserva `calling` para incidentes reales — el uso excesivo causa fatiga de alertas

## Preguntas frecuentes

**¿Direct es solo para desarrolladores?** No. Cualquier persona con una herramienta que pueda enviar solicitudes HTTP puede usarlo.

**¿Direct admite solicitudes GET?** Sí, pero se recomienda POST con JSON.

**¿Qué ocurre si mi suscripción caduca y uso `calling`?** Cae automáticamente a `time-sensitive`.
