# 如何把 QQ 邮箱的重要邮件转成电话通知

很多人并不是没收到重要 QQ 邮件，而是看晚了。开会、写代码、通勤时，普通推送太容易被淹没。更稳妥的做法是：只把关键邮件转发到 Echobell，让手机以“来电”方式提醒你。

这篇文章会带你从 0 到 1 完成配置。

先看这些文档会更顺：

- QQ 邮箱接入指南：[/zh/docs/email-to-call/qq](/zh/docs/email-to-call/qq)
- 邮件触发基础：[/zh/docs/email-trigger](/zh/docs/email-trigger)
- 通知类型（含 Calling）：[/zh/docs/notification](/zh/docs/notification)
- 模板与变量：[/zh/docs/template](/zh/docs/template)
- 条件过滤（可选兜底）：[/zh/docs/conditions](/zh/docs/conditions)

## 为什么要用电话通知？

- 来电提醒比普通推送更不容易错过
- 日常邮件保持安静，只有高优先级才“打电话”
- 语音播报让你不用盯屏幕也能理解告警内容

如果你要处理生产事故、客户升级、财务异常，这套方案非常实用。

## 准备工作

- 已安装 Echobell（[App Store](https://apps.apple.com/app/id6743597198)）
- 已创建一个用于紧急邮件的频道
- 可访问 QQ 邮箱设置

## 第一步 - 在 Echobell 建立“重要邮件”频道

1. 新建频道，例如 `QQ 关键邮件`。
2. 通知类型设置为 **Calling**。
3. 如需更稳妥送达，可在 App 设置中开启失败重试。

相关文档：[/zh/docs/notification](/zh/docs/notification)

## 第二步 - 复制频道邮箱并配置模板

在频道详情页复制专属邮箱地址。后面 QQ 邮箱规则会把重要邮件转发到这个地址。

建议模板保持简短，便于语音播报：

标题模板：

```txt
[QQ 重要邮件] {{subject}}
```

正文模板：

```txt
来自 {{from}}。{{text}}
```

模板进阶：[/zh/docs/template](/zh/docs/template)

## 第三步 - 在 QQ 邮箱配置转发规则

打开 QQ 邮箱设置，创建筛选/过滤规则，建议按以下条件筛选“重要邮件”：

- 发件人（老板、财务、安全、核心客户）
- 主题关键词（如 `紧急`、`发票`、`失败`、`验证`）
- 动作：转发到 Echobell 频道邮箱

首次启用转发时，QQ 邮箱会发送一封认证邮件。你可以在 Echobell 通知右上角找到链接，点击后查看邮件正文，再点正文里的认证链接即可完成启用。

这一步应作为主过滤层。为了隐私，优先在 QQ 邮箱侧完成筛选，只把必要邮件发到 Echobell。

建议先从一条严格规则开始，验证稳定后再扩展，避免通知轰炸。

QQ 专项步骤参考：[/zh/docs/email-to-call/qq](/zh/docs/email-to-call/qq)

## 第四步 - 可选兜底：在 Echobell 使用 Conditions

仅当 QQ 邮箱规则表达不了你的筛选逻辑时，再开启这层兜底过滤。

示例：

仅财务发件人触发：

```js
from == "billing@company.com"
```

主题包含 urgent 才触发：

```js
subject.toLowerCase().includes("urgent")
```

发件人 + 关键词组合：

```js
(from == "security@company.com" || from == "ops@company.com") && subject.toLowerCase().includes("alert")
```

条件语法：[/zh/docs/conditions](/zh/docs/conditions)

## 隐私与告知义务（务必说明）

在对外发布这套方案时，建议明确告知用户：

- 重要条件应优先在 QQ 邮箱中配置，尽量减少转发到 Echobell 的邮件范围。
- 用户需要知晓：被规则命中的邮件内容会被转发用于通知送达；应遵循团队制度与当地合规要求。

## 第五步 - 端到端测试

1. 在 QQ 邮箱发送一封符合规则的测试邮件。
2. 确认邮件已自动转发到频道邮箱。
3. 确认 Echobell 触发电话提醒。
4. 根据播报效果微调模板与规则。

## 常见问题排查

- 没接到电话：
  - 确认订阅通知类型是 Calling
  - 检查 iOS 通知权限与专注模式设置
- 电话太多：
  - 先收紧 QQ 规则
  - 若仍不足，再在 Echobell 增加 Conditions 兜底
- 语音内容太长：
  - 精简模板，只保留核心字段

## 下一步建议

- 按场景拆分频道（财务 / 安全 / 客户）
- 普通信息用推送，电话仅用于真正紧急事件
- 每月复盘并优化邮箱筛选规则
