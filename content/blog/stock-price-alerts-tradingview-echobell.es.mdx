# Alertas de precio de acciones con TradingView y Echobell

Las notificaciones push estándar son fáciles de perder cuando los mercados se mueven rápido. Esta guía cubre una configuración directa para enrutar alertas de precio de TradingView a llamadas telefónicas de Echobell o notificaciones urgentes, para que te avisen en el momento en que el precio alcanza tu nivel.

Documentación relevante:

- Qué es Echobell → [/docs/what-is-echobell](/en/docs/what-is-echobell)
- Resumen de webhooks → [/docs/webhook](/en/docs/webhook)
- Tipos de notificación (Normal, Urgente, Llamada) → [/docs/notification](/en/docs/notification)
- Plantillas y variables → [/docs/template](/en/docs/template)

## Requisitos previos

- Echobell instalado ([App Store](https://apps.apple.com/app/id6743597198))
- Cuenta de TradingView con soporte de webhook (Pro/Pro+/Premium)

## 1. Crear un canal en Echobell

1. En Echobell, crea un canal (p. ej. "Stocks: AAPL levels").
2. Configura el tipo de notificación. Para movimientos de precio urgentes usa **Calling**; de lo contrario, **Time Sensitive** ([/docs/notification](/en/docs/notification)).
3. Configura las plantillas para que la llamada lea contexto útil:

   Título: `{{ticker}} hit {{price}}`

   Cuerpo: `Rule: {{rule}} on {{ticker}} at {{price}}`

4. Copia la URL del Webhook del canal y mantenla privada ([/docs/webhook](/en/docs/webhook)).

## 2. Configurar la alerta en TradingView

1. Crea una alerta en tu gráfico para la condición que deseas (p. ej. el cierre cruza un nivel de precio).
2. Activa **Webhook URL** y pega la URL de Echobell.
3. Establece el mensaje en JSON para que Echobell pueda sustituir variables a través de tu plantilla:

```json
{
  "ticker": "{{ticker}}",
  "price": "{{close}}",
  "rule": "Close crossed 150",
  "externalLink": "https://www.tradingview.com/chart/"
}
```

Notas:
- Mantén las claves JSON simples y que coincidan con las variables de tu plantilla
- `externalLink` es opcional; Echobell lo muestra como enlace en el registro de la notificación ([/docs/notification](/en/docs/notification))

## 3. Probar el flujo

Activa la alerta o usa una condición de prueba temporal. Deberías recibir una llamada (o notificación urgente) con el contenido de tu plantilla. Ajusta la redacción hasta que sea clara e inequívoca al escucharla en una llamada.

## Buenas prácticas

- Usa **Calling** solo para niveles de precio críticos; usa **Time Sensitive** para todo lo demás
- Crea canales separados por símbolo o estrategia para controlar el ruido
- Incluye el marco temporal en tu plantilla si es relevante (p. ej. "cierre de 15m cruzó 150")
