# Alertes de prix boursiers avec TradingView et Echobell

Les notifications push standard sont faciles à manquer quand les marchés bougent vite. Ce guide couvre une configuration simple pour acheminer les alertes de prix TradingView vers des appels téléphoniques Echobell ou des notifications urgentes, afin d'être notifié dès que le prix atteint votre niveau.

Documentation utile :

- Qu'est-ce qu'Echobell → [/docs/what-is-echobell](/en/docs/what-is-echobell)
- Présentation des Webhooks → [/docs/webhook](/en/docs/webhook)
- Types de notifications (Normal, Urgent, Appel) → [/docs/notification](/en/docs/notification)
- Modèles et variables → [/docs/template](/en/docs/template)

## Prérequis

- Echobell installé ([App Store](https://apps.apple.com/app/id6743597198))
- Compte TradingView avec support Webhook (Pro/Pro+/Premium)

## 1. Créer un canal Echobell

1. Dans Echobell, créez un canal (ex. « Stocks : niveaux AAPL »).
2. Réglez le type de notification. Pour les mouvements de prix urgents, utilisez **Appel** ; sinon **Urgent** ([/docs/notification](/en/docs/notification)).
3. Configurez les modèles pour que l'appel lise un contexte utile :

   Titre : `{{ticker}} hit {{price}}`

   Corps : `Rule: {{rule}} on {{ticker}} at {{price}}`

4. Copiez l'URL Webhook du canal et gardez-la privée ([/docs/webhook](/en/docs/webhook)).

## 2. Configurer l'alerte TradingView

1. Créez une Alerte sur votre graphique pour la condition souhaitée (ex. la clôture croise un niveau de prix).
2. Activez **Webhook URL** et collez l'URL Echobell.
3. Définissez le message en JSON pour qu'Echobell puisse substituer les variables via votre modèle :

```json
{
  "ticker": "{{ticker}}",
  "price": "{{close}}",
  "rule": "Close crossed 150",
  "externalLink": "https://www.tradingview.com/chart/"
}
```

Notes :
- Gardez les clés JSON simples et correspondant à vos variables de modèle
- `externalLink` est optionnel ; Echobell l'affiche comme lien cliquable dans l'enregistrement de la notification ([/docs/notification](/en/docs/notification))

## 3. Tester le flux

Déclenchez l'alerte ou utilisez une condition de test temporaire. Vous devriez recevoir un appel (ou une notification urgente) qui lit le contenu de votre modèle. Ajustez la formulation jusqu'à ce qu'elle soit claire et sans ambiguïté à l'écoute.

## Bonnes pratiques

- Utilisez **Appel** uniquement pour les niveaux de prix critiques ; utilisez **Urgent** pour tout le reste
- Créez des canaux séparés par symbole ou stratégie pour contrôler le bruit
- Incluez l'unité de temps dans votre modèle si elle est pertinente (ex. « clôture 15m a croisé 150 »)
