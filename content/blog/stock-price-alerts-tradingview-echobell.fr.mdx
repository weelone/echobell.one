# Des alertes de prix impossibles à manquer : TradingView → Echobell

Les marchés bougent vite. Avec des notifications classiques, on finit par rater l’instant. Voici une configuration simple pour transformer des alertes de prix TradingView en appels Echobell (ou notifications Time Sensitive) afin d’agir quand le prix atteint votre niveau.

Ressources utiles :

- Qu’est‑ce qu’Echobell → [/fr/docs/what-is-echobell](/fr/docs/what-is-echobell)
- Webhooks → [/fr/docs/webhook](/fr/docs/webhook)
- Types de notification → [/fr/docs/notification](/fr/docs/notification)
- Templates et variables → [/fr/docs/template](/fr/docs/template)

## Prérequis

- Echobell installé et compte créé ([App Store](https://apps.apple.com/app/id6743597198))
- Compte TradingView (webhooks généralement réservés Pro/Pro+/Premium)

## 1) Créer un canal Echobell

1. Dans Echobell, créez un canal (ex. « Stocks : niveaux AAPL »).
2. Choisissez le type. Urgent → **Calling**, sinon **Time Sensitive**. Voir [/fr/docs/notification](/fr/docs/notification).
3. Définissez les Templates pour un message vocal clair :

   Titre : `{{ticker}} a atteint {{price}}`

   Corps : `Règle : {{rule}} sur {{ticker}} à {{price}}`

4. Copiez l’URL de Webhook du canal (gardez‑la privée). Détails : [/fr/docs/webhook](/fr/docs/webhook)

## 2) Configurer l’alerte TradingView

1. Créez une alerte sur le graphique (ex. close croise un prix).
2. Activez **Webhook URL** et collez l’URL Echobell.
3. Utilisez un message JSON pour transmettre des variables :

```json
{
  "ticker": "{{ticker}}",
  "price": "{{close}}",
  "rule": "La clôture a croisé 150",
  "externalLink": "https://www.tradingview.com/chart/"
}
```

Conseils :

- Clés simples (`ticker`, `price`, `rule`).
- `externalLink` est optionnel ; Echobell l’affiche dans l’historique local. Voir [/fr/docs/notification](/fr/docs/notification).

## 3) Tester le flux

Déclenchez l’alerte ou utilisez une condition temporaire. Vous devriez recevoir un appel (ou une alerte Time Sensitive) lisant votre titre/corps. Affinez le texte jusqu’à ce que ce soit limpide.

## Bonnes pratiques

- Réservez les appels aux niveaux vraiment urgents ; sinon Time Sensitive.
- Séparez les canaux par symbole ou stratégie pour réduire le bruit.
- Ajoutez l’unité de temps dans le template si pertinent (ex. « clôture 15m »).
