# 用 TradingView 和 Echobell 设置股价告警

行情快速波动时，普通推送通知很容易被忽略。本文介绍如何将 TradingView 价格告警路由到 Echobell 来电或时间敏感通知，让价格触达你的目标位时第一时间响应。

相关文档：

- 什么是 Echobell → [/docs/what-is-echobell](/en/docs/what-is-echobell)
- Webhooks 概览 → [/docs/webhook](/en/docs/webhook)
- 通知类型（普通、时间敏感、来电）→ [/docs/notification](/en/docs/notification)
- 模板与变量 → [/docs/template](/en/docs/template)

## 前提条件

- 已安装 Echobell（[App Store](https://apps.apple.com/app/apple-store/id6743597198?pt=128151925&ct=blog-stock-price-alerts-tradingvi-1ol2kk&mt=8)）
- 拥有支持 Webhook 的 TradingView 账户（Pro/Pro+/Premium）

## 1. 创建 Echobell 频道

1. 在 Echobell 中创建频道（如"Stocks: AAPL levels"）。
2. 设置通知类型。紧急价格变动用**来电**，否则用**时间敏感**（参见 [/docs/notification](/en/docs/notification)）。
3. 配置模板，让来电内容提供有用的上下文：

   标题：`{{ticker}} hit {{price}}`

   正文：`Rule: {{rule}} on {{ticker}} at {{price}}`

4. 复制频道的 Webhook URL 并妥善保管（参见 [/docs/webhook](/en/docs/webhook)）。

## 2. 配置 TradingView 告警

1. 在图表上创建告警，设置触发条件（如收盘价穿越某价位）。
2. 启用 **Webhook URL**，粘贴 Echobell 的 URL。
3. 将消息设置为 JSON 格式，Echobell 可通过模板进行变量替换：

```json
{
  "ticker": "{{ticker}}",
  "price": "{{close}}",
  "rule": "Close crossed 150",
  "externalLink": "https://www.tradingview.com/chart/"
}
```

注意：
- JSON 键名保持简单，与模板变量一致
- `externalLink` 可选；Echobell 会在通知记录中以可点击链接的形式展示（参见 [/docs/notification](/en/docs/notification)）

## 3. 测试全流程

触发告警或使用临时测试条件。你应该会收到来电（或时间敏感通知），内容为你设置的模板文字。调整措辞，直到通话内容清晰无歧义。

## 最佳实践

- **来电**仅用于关键价位；其他情况使用**时间敏感**
- 按标的或策略创建独立频道，控制噪音
- 如有需要，在模板中加入时间周期（如"15m close crossed 150"）
