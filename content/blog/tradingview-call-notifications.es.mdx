# Recibe alertas de TradingView como llamadas telefónicas con Echobell

Las notificaciones push estándar son fáciles de perder cuando una operación se está desarrollando. Esta guía muestra cómo conectar TradingView a Echobell via webhook para que las alertas de precio activen llamadas telefónicas con contenido de voz personalizable.

Documentación relevante:

- Qué es Echobell → [/docs/what-is-echobell](/en/docs/what-is-echobell)
- Resumen de webhooks → [/docs/webhook](/en/docs/webhook)
- Tipos de notificación (Normal, Urgente, Llamada) → [/docs/notification](/en/docs/notification)
- Plantillas y variables → [/docs/template](/en/docs/template)

## Requisitos previos

- App de Echobell instalada ([App Store](https://apps.apple.com/app/id6743597198))
- Cuenta de TradingView con soporte de webhook (Pro/Pro+/Premium)

## Paso 1 — Crear un canal en Echobell

1. Abre Echobell → Crear Canal. Nómbralo algo como "TradingView Calls".
2. Configura el tipo de notificación. Para alertas urgentes usa **Calling**; para las menos críticas usa **Time Sensitive** o **Normal** ([/docs/notification](/en/docs/notification)).
3. Opcionalmente, configura plantillas para que la llamada lea datos útiles. Ejemplo:

   Título: `{{ticker}} crossed {{price}}`

   Cuerpo: `Signal: {{action}} at {{price}} on {{ticker}}`

   Referencia completa de plantillas: [/docs/template](/en/docs/template)

4. Copia la URL del Webhook del canal desde la configuración. Mantenla privada. Detalles: [/docs/webhook](/en/docs/webhook)

## Paso 2 — Configurar la alerta en TradingView

1. Abre tu gráfico y crea o edita una alerta.
2. Define la condición (nivel de precio, indicador, señal de estrategia, etc.).
3. En la sección Notifications, activa **Webhook URL** y pega tu webhook de Echobell.
4. Establece el payload del mensaje en JSON para que Echobell pueda referenciar variables en tu plantilla:

```json
{
  "ticker": "{{ticker}}",
  "price": "{{close}}",
  "action": "{{strategy.order.action}}",
  "externalLink": "https://www.tradingview.com/chart/"
}
```

Notas:
- Usa las variables integradas de TradingView (`{{ticker}}`, `{{close}}`, etc.)
- `externalLink` es opcional; cuando está presente, Echobell lo muestra como enlace en el registro de la notificación ([/docs/notification](/en/docs/notification))

## Paso 3 — Probar el flujo completo

Activa la alerta o usa una condición de prueba. En segundos deberías recibir una llamada con el contenido de tu plantilla. Si la llamada resulta demasiado intrusiva para señales de baja prioridad, cambia el canal a **Time Sensitive** y usa **Calling** solo para niveles críticos.

## Solución de problemas

- **No llega ninguna llamada**: confirma que el tipo de notificación del canal está configurado como Calling y que la app tiene permisos de notificación
- **Error 4xx en el webhook de TradingView**: verifica la URL exacta del webhook de Echobell y que el token del canal esté vigente
- **Las variables no aparecen en la llamada**: confirma que las claves JSON coinciden exactamente con los nombres de variables en la plantilla (distingue mayúsculas de minúsculas)
- **Demasiadas llamadas**: reduce la prioridad a Time Sensitive o ajusta las condiciones de alerta en TradingView

## Consejos para plantillas

Una plantilla clara hace que la llamada sea inmediatamente accionable:

- `{{ticker}} crossed {{price}} on 15m timeframe.`
- `Action: {{action}}.`
- Incluye `externalLink` para un acceso rápido al gráfico desde el registro de la notificación

Documentación completa de plantillas: [/docs/template](/en/docs/template)
