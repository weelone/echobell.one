# Ne manquez plus un trade : transformez vos alertes TradingView en appels avec Echobell

Si vous avez déjà raté un setup parfait parce qu’une notification s’est perdue, vous n’êtes pas seul. Je voulais quelque chose d’impossible à ignorer quand le prix touchait mes niveaux—alors j’ai transformé mes alertes TradingView en vrais appels téléphoniques avec Echobell. La mise en place prend quelques minutes et la sérénité est réelle.

Dans ce guide, on relie TradingView à Echobell via un webhook, on personnalise le texte vocal et on teste le tout de bout en bout.

## Pourquoi des appels pour TradingView ?

- Les appels passent à travers les modes de concentration quand les secondes comptent
- Dans Echobell, vous choisissez la priorité par canal (Normal, Sensible au temps, Appel)
- Les modèles transforment des données brutes en un script clair que vous entendrez durant l’appel

Note SEO : cet article traite des alertes TradingView en appels téléphoniques via l’intégration webhook d’Echobell.

## Prérequis

- Application Echobell installée et compte créé ([App Store](https://apps.apple.com/app/id6743597198))
- Compte TradingView (webhooks généralement réservés aux offres Pro/Pro+/Premium)

## Étape 1 — Créez un canal Echobell (et activez Appel)

1. Dans l’app Echobell → Créer un canal. Nommez‑le « TradingView Calls » et choisissez une couleur reconnaissable.

2. Choisissez le type de notification. Pour l’urgent, sélectionnez **Appel**. Pour le reste, **Sensible au temps** ou **Normal**. Voir [/fr/docs/notification](/fr/docs/notification).

3. Personnalisez le contenu (recommandé). L’appel peut dire exactement ce qui s’est passé. Associez‑le aux variables TradingView :

   Titre : `{{ticker}} a franchi {{price}}`

   Corps : `Signal : {{action}} à {{price}} sur {{ticker}}`

Plus d’infos sur les variables/expressions : [/fr/docs/template](/fr/docs/template)

4. Copiez l’URL de Webhook du canal (détails du canal). C’est l’adresse appelée par TradingView. Traitez‑la comme un secret. Détails : [/fr/docs/webhook](/fr/docs/webhook)

## Étape 2 — Connectez TradingView à Echobell

1. Dans TradingView, ouvrez votre graphique et créez/modifiez une alerte.

2. Définissez la condition (prix, indicateur, signal de stratégie, etc.).

3. Dans Notifications/Actions, activez **Webhook URL** et collez votre webhook Echobell (Étape 1).

4. Rédigez le Message en JSON pour transmettre des valeurs dynamiques que Echobell restituera pendant l’appel :

```json
{
  "ticker": "{{ticker}}",
  "price": "{{close}}",
  "action": "{{strategy.order.action}}",
  "externalLink": "https://www.tradingview.com/chart/"
}
```

Notes :

- Utilisez les variables TradingView (ex. `{{ticker}}`, `{{close}}`).
- `externalLink` est optionnel : s’il est présent, Echobell l’affiche dans l’historique local. Voir [/fr/docs/notification](/fr/docs/notification).

## Étape 3 — Testez de bout en bout

Déclenchez l’alerte (ou utilisez un test). En quelques secondes, vous devriez recevoir un appel Echobell avec votre contenu personnalisé. Pour un mode moins intrusif, essayez d’abord Sensible au temps et passez à Appel pour l’urgent.

## Dépannage

- Pas d’appel ? Vérifiez que le type « Appel » est sélectionné et que l’app a les autorisations.
- Erreur 4xx sur le webhook ? Vérifiez l’URL exacte et que le jeton du canal n’a pas été régénéré.
- Variables manquantes ? Les clés JSON doivent correspondre exactement aux variables de modèle (sensible à la casse).
- Trop d’appels ? Baissez la priorité à Sensible au temps ou affinez les conditions TradingView.

## Faites briller vos modèles

Les modèles transforment les données en un script clair pour l’appel. Idées :

- Contexte : « {{ticker}} a franchi {{price}} en 15 min »
- Direction : « Action : {{action}} »
- Lien `externalLink` pour revoir plus tard dans l’historique local

Toutes les options ici → [/fr/docs/template](/fr/docs/template)
