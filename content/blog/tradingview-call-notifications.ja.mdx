# もう取引の機会を逃さない：TradingView のアラートを Echobell で電話通知に

完璧なセットアップを逃してしまう原因は、通知を見落とすことにあります。私は「絶対に気づける仕組み」が欲しくて、TradingView のアラートを Echobell の電話通知に変えました。数分でつながり、安心感が違います。

このガイドでは、Webhook で TradingView と Echobell を接続し、通話内容をテンプレートでカスタマイズし、エンドツーエンドでテストします。

全体像のためのリンク：

- Echobell とは → [/ja/docs/what-is-echobell](/ja/docs/what-is-echobell)
- Webhook の概要 → [/ja/docs/webhook](/ja/docs/webhook)
- 通知タイプ（通常・時間重視・通話） → [/ja/docs/notification](/ja/docs/notification)
- テンプレートと変数 → [/ja/docs/template](/ja/docs/template)

## なぜ「通話」通知？

- 秒単位で重要なときでも、通話はフォーカスモードを貫通します
- Echobell ではチャンネルごとに優先度（通常/時間重視/通話）を選べます
- テンプレートでデータを「聞き取りやすい文章」にできます

SEO メモ：本記事は、Echobell の Webhook 連携で TradingView のアラートを電話通知にする方法を解説します。

## 前提条件

- Echobell アプリのインストールとアカウント作成（[App Store](https://apps.apple.com/app/id6743597198)）
- TradingView アカウント（Webhook は Pro/Pro+/Premium で利用可能）

## ステップ 1 — Echobell チャンネル作成（通話を選択）

1）Echobell アプリ → チャンネル作成。名称は「TradingView Calls」など、ひと目で分かるものに。

2）通知タイプを選択。緊急なら**通話**、それ以外は**時間重視**や**通常**も可。→ [/ja/docs/notification](/ja/docs/notification)

3）テンプレートを調整（推奨）。通話で「何が起きたか」をそのまま伝えられます。TradingView の変数と組み合わせましょう：

    タイトル：`{{ticker}} が {{price}} をクロス`

    本文：`シグナル: {{action}} @ {{price}} / {{ticker}}`

変数と式の詳細 → [/ja/docs/template](/ja/docs/template)

4）チャンネル詳細から Webhook URL をコピー。TradingView が呼び出す先です。秘密として扱ってください。→ [/ja/docs/webhook](/ja/docs/webhook)

## ステップ 2 — TradingView と接続

1）TradingView でチャートを開き、アラートを作成/編集。

2）条件（価格・インジ・ストラテジーなど）を設定。

3）通知/アクションで**Webhook URL**を有効にし、ステップ 1 の URL を貼り付け。

4）メッセージは JSON で。動的値を渡すと、Echobell が通話で読み上げできます：

```json
{
  "ticker": "{{ticker}}",
  "price": "{{close}}",
  "action": "{{strategy.order.action}}",
  "externalLink": "https://www.tradingview.com/chart/"
}
```

補足：

- TradingView の変数（例：`{{ticker}}`, `{{close}}`）を使います。
- `externalLink`は任意。ローカル記録にリンクとして表示されます → [/ja/docs/notification](/ja/docs/notification)

## ステップ 3 — 動作確認

アラートを発火（テストでも可）。数秒以内に、設定した内容で Echobell の電話が鳴るはずです。まずは時間重視で試し、本当に緊急なものだけ通話にするのもおすすめ。

## トラブルシューティング

- 通話が来ない：通知タイプが通話か、通知権限があるか確認
- Webhook が 4xx：URL の誤り、チャンネルトークンのローテーションを確認
- 変数が反映されない：JSON キーとテンプレート変数の表記（大文字小文字）を一致させる
- 通話が多すぎる：時間重視に落とす、または TradingView の条件を調整

## テンプレートを活かすコツ

通話で聞いた瞬間に理解できる文章を：

- 文脈を追加：`{{ticker}} が {{price}}（15 分足）`
- 方向性を含める：`アクション：{{action}}`
- `externalLink`で後から記録からチャートへ

詳しくは → [/ja/docs/template](/ja/docs/template)
