# 不再错过交易：用 Echobell 将 TradingView 警报变成电话通知

完美的入场从指缝溜走，往往只是因为通知没被注意到。我也经历过，所以把 TradingView 的警报改造成了 Echobell 的“电话通知”。几分钟即可接通，心态踏实很多。

本文演示如何通过 Webhook 将 TradingView 连接到 Echobell，如何用模板自定义通话内容，并完成端到端测试。

建议先阅读：

- 什么是 Echobell → [/zh/docs/what-is-echobell](/zh/docs/what-is-echobell)
- Webhook 概览 → [/zh/docs/webhook](/zh/docs/webhook)
- 通知与优先级（普通/时间敏感/电话） → [/zh/docs/notification](/zh/docs/notification)
- 模板与变量 → [/zh/docs/template](/zh/docs/template)

## 为什么选择“电话通知”？

- 秒级场景中，电话能穿透专注模式和环境噪音
- Echobell 可按频道选择优先级（普通、时间敏感、电话）
- 模板可把数据转换成“听得懂”的语句

SEO 说明：本文讲解用 Echobell 的 Webhook 集成把 TradingView 警报变成电话通知。

## 前置条件

- 安装 Echobell 并完成账号设置（[App Store](https://apps.apple.com/app/id6743597198)）
- TradingView 账号（Webhook 通常需 Pro/Pro+/Premium）

## 步骤 1 — 创建 Echobell 频道（选择“电话”）

1）在 Echobell 应用中 → 创建频道。建议命名为“TradingView Calls”等，并选一个醒目的颜色。

2）选择通知类型。紧急场景请选择 **电话**；不那么紧急的可用 **时间敏感** 或 **普通**。详见 [/zh/docs/notification](/zh/docs/notification)。

3）在“模板”中自定义内容（推荐）。通话可以直接描述所发生的事件。结合 TradingView 变量：

标题：`{{ticker}} 突破 {{price}}`

正文：`信号：{{action}} @ {{price}} / {{ticker}}`

变量与表达式 → [/zh/docs/template](/zh/docs/template)

4）从频道详情复制 Webhook URL。TradingView 将调用该地址。请妥善保管。详情见：[/zh/docs/webhook](/zh/docs/webhook)

## 步骤 2 — 连接 TradingView 与 Echobell

1）在 TradingView 中打开图表，新建/编辑一个警报。

2）设置触发条件（价格、指标、策略信号等）。

3）在“通知/操作”处勾选 **Webhook URL**，粘贴步骤 1 的地址。

4）将消息写成 JSON，传入动态值，这些值可在通话中被 Echobell 模板“读出”：

```json
{
  "ticker": "{{ticker}}",
  "price": "{{close}}",
  "action": "{{strategy.order.action}}",
  "externalLink": "https://www.tradingview.com/chart/"
}
```

提示：

- 使用 TradingView 变量（如 `{{ticker}}`, `{{close}}`）。
- `externalLink` 可选：用于在本地记录中展示链接 → [/zh/docs/notification](/zh/docs/notification)

## 步骤 3 — 测试全流程

触发该警报（或用测试警报）。几秒内，您应能接到带有自定义内容的 Echobell 来电。如果想先温和一点，可先用“时间敏感”，真正紧急时再切换到“电话”。

## 常见问题排查

- 没有来电？检查频道的通知类型是否为“电话”，以及应用是否有通知权限。
- Webhook 4xx？请核对 URL 是否准确，或频道 Token 是否被重置。
- 模板未替换？确保 JSON 键名与模板变量完全一致（区分大小写）。

## 用模板，让内容更“会说话”

建议在通话中加入足量上下文：

- 场景说明：「{{ticker}} 于 15 分钟级别触达 {{price}}」
- 方向动作：「动作：{{action}}」
- 通过 `externalLink` 方便事后从记录跳回图表

更多模板玩法 → [/zh/docs/template](/zh/docs/template)
