# Faites sonner vos alertes Upptime avec Echobell

Upptime surveille déjà vos sites via GitHub Actions, mais quand ça casse il faut un signal qui tranche. Echobell transforme ces vérifications en pushes sensibles au temps ou en appels, sans serveur supplémentaire : un webhook par canal suffit.

## Pourquoi ce duo fonctionne

- **Escalade réelle** : basculez en appels pour les endpoints qui ne peuvent pas attendre.  
- **Contexte structuré** : chaque alerte inclut site, statut, code de réponse et lien vers l’issue GitHub.  
- **Effort minimal** : deux secrets dans le dépôt Upptime, le reste de la config reste intact.

## Activation en quelques minutes

1) Dans Echobell, copiez le webhook du canal : `https://hook.echobell.one/t/<channel-token>` (standard, sensible au temps ou appel).  
2) Dans votre dépôt Upptime, ajoutez ces secrets GitHub Actions :

```
NOTIFICATION_CUSTOM_WEBHOOK=true
NOTIFICATION_CUSTOM_WEBHOOK_URL=https://hook.echobell.one/t/<channel-token>?site=$SITE_NAME&status=$STATUS&url=$SITE_URL&issue=$ISSUE_URL&code=$RESPONSE_CODE&emoji=$EMOJI
NOTIFICATIONS_DOWN_MESSAGE=$EMOJI $SITE_NAME is $STATUS ($RESPONSE_CODE) – $ISSUE_URL
NOTIFICATIONS_UP_MESSAGE=$EMOJI $SITE_NAME is back up – $SITE_URL
```

3) Dans Echobell, définissez un gabarit simple (`{{site}}`, `{{status}}`, `{{code}}`, `{{url}}`, `{{issue}}`) et choisissez si le canal doit appeler ou rester en push.

C’est tout : Upptime fournit les données, Echobell gère le style de notification. Pour chaque détail, voyez le [guide de configuration](/fr/docs/developer/upptime), et pour les réglages Upptime reportez-vous à la [documentation officielle](https://upptime.js.org/docs/notifications).

## Ce que vous y gagnez

- Détection plus rapide des dégradations ou pannes  
- Liens directs vers le site concerné et l’issue GitHub  
- Un chemin d’escalade fiable sans toucher au reste de votre configuration Upptime

Testez sur une URL surveillée, provoquez une panne de test et vous verrez (ou entendrez) l’alerte immédiatement.
