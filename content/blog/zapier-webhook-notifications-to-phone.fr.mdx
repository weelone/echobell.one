# Envoyez les notifications webhook Zapier sur votre telephone avec Echobell

Zapier est excellent pour connecter des applications. Ce qu il ne decide pas pour vous, c est le niveau d urgence que devrait avoir un evenement une fois arrive sur le telephone.

Cela compte beaucoup pour des cas comme :

- un lead VIP qui envoie un formulaire
- un echec de paiement ou de checkout
- un ticket de support marque urgent
- une opportunite CRM qui demande une action immediate
- un workflow no-code qui detecte une panne de service

Dans ces cas, un email ou un simple push est souvent trop faible. Echobell ameliore la derniere etape avec :

- notifications standard
- alertes time-sensitive
- alertes par appel

## A quoi ressemble le montage

Le schema est simple :

1. Zapier detecte un evenement.
2. Le Zap envoie un webhook vers un canal Echobell.
3. Echobell transforme ce payload en notification ou en appel.

Vous ne remplacez pas Zapier. Vous ameliorez seulement la livraison vers le telephone.

## Etape 1 : creez un canal Echobell

Dans Echobell :

1. Creez un nouveau canal
2. Donnez-lui un nom lie au workflow, par exemple `Support VIP`, `Escalades commerciales` ou `Echecs de paiement`
3. Copiez l URL webhook du canal

Pour les bases, voyez [webhook notifications](/en/features/webhooks).

## Etape 2 : ajoutez une action Webhooks dans Zapier

Dans le Zap :

1. Choisissez le trigger habituel
2. Ajoutez `Webhooks by Zapier`
3. Utilisez `POST`
4. Collez l URL webhook Echobell

Puis mappez les champs utiles dans le body JSON.

Exemple pour une escalation support urgente :

```json
{
  "title": "Ticket urgent de {{customer_name}}",
  "message": "{{subject}}",
  "severity": "{{priority}}",
  "ticketId": "{{ticket_id}}",
  "externalLink": "{{ticket_url}}"
}
```

Et dans Echobell :

```txt
Title: {{title}}
Body: {{message}} (Priority: {{severity}})
```

## Etape 3 : choisissez la bonne urgence

C est la partie que beaucoup de tutoriels oublient.

- utilisez le mode standard pour les automatisations courantes
- utilisez time-sensitive pour les workflows qui doivent passer Focus Mode
- utilisez les appels quand manquer l alerte coute cher

Bons candidats a une urgence plus forte :

- fraude ou panne de paiement
- escalations support urgentes
- pannes detectees par un workflow no-code
- formulaires qui demandent un rappel immediat

Si votre vrai probleme est Focus Mode, poursuivez avec [Comment contourner iOS Focus Mode pour les alertes critiques](/en/blog/how-to-bypass-ios-focus-mode-for-critical-alerts).

## Workflows Zapier qui marchent tres bien avec Echobell

### 1. Leads urgents

Typeform, Tally, HubSpot ou formulaires personnalises peuvent devenir des alertes directes pour les ventes ou les founders.

### 2. Alertes paiement et checkout

Des evenements Stripe, Shopify ou de billing interne meritent souvent mieux qu un email.

### 3. Escalation support

Help Scout, Zendesk, Intercom ou des regles Gmail peuvent envoyer les incidents urgents directement au bon telephone.

### 4. Operations internes

Zapier peut observer des outils SaaS, des feuilles de calcul ou des systemes no-code, puis laisser Echobell gerer la livraison.

## Bonnes pratiques

### Un canal par workflow

Ne mettez pas toutes les automatisations dans le meme canal.

Exemples :

- `Leads urgents`
- `Incidents revenu`
- `Escalades support`
- `Operations internes`

### Ajoutez un lien vers la source

Si votre payload contient `externalLink`, Echobell peut ramener le destinataire plus vite vers le ticket, le deal ou l enregistrement d origine.

### Gardez les appels pour les echecs couteux

Si chaque Zap fait sonner le telephone, plus personne ne fera confiance au signal.

## Recommandation finale

Zapier sait deja tres bien sortir les donnees des applications. La vraie question est de savoir si ces evenements arrivent avec la bonne urgence sur le telephone.

Si vous avez ne serait-ce qu une automatisation trop importante pour un email ou un push enterre, faites-la passer par Echobell :

- [Phone call alerts for critical incidents](/en/features/call-notifications)
