# Echobell で Zapier の Webhook 通知をスマホに送る

Zapier はアプリ同士をつなぐのが得意です。ただし、イベントがスマホに届く時に「どれくらい緊急であるべきか」は決めてくれません。

その差が大きく出るのは、たとえば次のような時です。

- VIP リードがフォームを送信した
- 支払いまたは checkout が失敗した
- サポートチケットが緊急扱いになった
- CRM の案件が今すぐ動くべき段階に入った
- ノーコードのフローがサービス障害を検知した

このようなイベントに対して、普通のメールや push だけでは弱いことがあります。Echobell は最後の配信レイヤーを次のように強化します。

- 標準通知
- time-sensitive 通知
- 電話アラート

## 全体像

流れはとてもシンプルです。

1. Zapier がイベントを検知する
2. Zap が Echobell のチャンネルに Webhook を送る
3. Echobell がそれを通知または電話に変える

つまり、Zapier はそのままで、最後の配信だけを強くできます。

## ステップ 1: Echobell でチャンネルを作る

Echobell で：

1. 新しいチャンネルを作成
2. `VIP Support`、`Sales Escalations`、`Payment Failures` のようにワークフロー単位で名前を付ける
3. チャンネルの Webhook URL をコピーする

基本を先に見たい場合は [Webhook notifications](/features/webhooks) を見てください。

## ステップ 2: Zapier に Webhooks アクションを追加する

Zap の中で：

1. 通常どおりトリガーを選ぶ
2. `Webhooks by Zapier` を追加
3. `POST` を選択
4. Echobell の Webhook URL を貼り付ける

その後、必要なフィールドを JSON body にマッピングします。

サポートの緊急エスカレーションなら、たとえばこうです。

```json
{
  "title": "Urgent support ticket from {{customer_name}}",
  "message": "{{subject}}",
  "severity": "{{priority}}",
  "ticketId": "{{ticket_id}}",
  "externalLink": "{{ticket_url}}"
}
```

Echobell 側のテンプレートはこう書けます。

```txt
Title: {{title}}
Body: {{message}} (Priority: {{severity}})
```

## ステップ 3: 本当に必要な緊急度を決める

ここが多くの Zapier チュートリアルで省かれる部分です。

- 日常的な自動化は標準通知
- Focus Mode を越えてほしいものは time-sensitive
- 見逃しコストが高いものは電話アラート

高い緊急度が向いている例：

- 不正検知や支払い失敗
- 緊急サポートエスカレーション
- ノーコードワークフローが見つけた障害
- すぐ折り返しが必要なフォーム送信

Focus Mode の突破が主目的なら、[重要アラートを iOS Focus Mode 越しに届ける方法](/blog/how-to-bypass-ios-focus-mode-for-critical-alerts) も役立ちます。

## Echobell と相性が良い Zapier ワークフロー

### 1. 重要リード通知

Typeform、Tally、HubSpot、独自フォームなどの送信を、営業や創業者のスマホへすぐ届けられます。

### 2. 支払い・checkout アラート

Stripe、Shopify、社内 billing の失敗通知は、メールより強い配信が向いています。

### 3. サポートエスカレーション

Help Scout、Zendesk、Intercom、Gmail ルールなどから、正しい担当者へ直接送れます。

### 4. 内部オペレーション

Zapier が SaaS、スプレッドシート、ノーコードシステムを監視し、Echobell が最後の配信を担当します。

## ベストプラクティス

### ワークフローごとにチャンネルを分ける

すべての Zap を一つの流れに入れないでください。

例：

- `Urgent leads`
- `Revenue incidents`
- `Support escalations`
- `Internal ops`

### 元データへのリンクを入れる

payload に `externalLink` を含めると、受信者は元のチケットや案件へ素早く戻れます。

### 電話は高コスト障害だけに使う

すべての Zap が電話を鳴らすようになると、誰もその信号を信頼しなくなります。

## 最後のおすすめ

Zapier はアプリからデータを取り出すのが得意です。本当の問題は、そのイベントが正しい緊急度でスマホに届くかどうかです。

メールや埋もれた push では弱い自動化が 1 つでもあるなら、Echobell に流してください。

- [Webhook notifications for iPhone](/features/webhooks)
- [Phone call alerts for critical incidents](/features/call-notifications)
- [Server down phone call alerts](/server-down-phone-call-alerts)

この小さな変更は、自動化ロジックをもう一段重ねるより大きな効果を生むことがよくあります。
