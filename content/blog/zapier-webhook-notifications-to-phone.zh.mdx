# 用 Echobell 把 Zapier Webhook 通知发到手机上

Zapier 很擅长连接应用和自动化流程，但它并不会替你决定“这个事件到底该以多高优先级打到手机上”。

当 Zap 处理的是下面这些事情时，这个差别就会很明显：

- VIP 线索提交了表单
- 支付或结账流程失败
- 支持系统把工单标记为紧急
- CRM 里的 deal 进入了需要立刻跟进的阶段
- 一个无代码流程检测到了服务异常

这时，普通邮件或普通推送通常都不够。Echobell 能把 Zapier 的最后一公里升级成：

- 普通通知
- 时间敏感通知
- 电话提醒

## 整体思路

模式很简单：

1. Zapier 检测到事件。
2. Zap 把一个 Webhook 请求发送到 Echobell 频道。
3. Echobell 把这个 payload 转成通知或电话提醒。

这意味着你不需要替换 Zapier，只需要把交付层做得更强。

## 第一步：创建 Echobell 频道

在 Echobell 中：

1. 创建一个新频道
2. 按工作流命名，例如 `VIP 支持`、`销售升级`、`支付失败`
3. 复制频道的 Webhook URL

如果你想先了解基础能力，可以看 [Webhook 通知](/en/features/webhooks)。

## 第二步：在 Zapier 中添加 Webhooks 动作

在 Zap 里：

1. 照常选择触发器
2. 添加 `Webhooks by Zapier`
3. 选择 `POST`
4. 粘贴 Echobell 的 Webhook URL

然后把有价值的字段映射进 JSON body。

例如，一个高优先级支持升级可以这样发：

```json
{
  "title": "来自 {{customer_name}} 的紧急支持工单",
  "message": "{{subject}}",
  "severity": "{{priority}}",
  "ticketId": "{{ticket_id}}",
  "externalLink": "{{ticket_url}}"
}
```

在 Echobell 里，频道模板可以写成：

```txt
Title: {{title}}
Body: {{message}} (Priority: {{severity}})
```

## 第三步：决定它到底有多紧急

很多 Zapier 教程都会跳过这一点，但这才是最关键的部分。

- 对常规自动化，使用普通通知
- 对需要突破专注模式的工作流，使用时间敏感通知
- 对错过代价很高的事件，使用电话提醒

适合高优先级的例子包括：

- 欺诈或支付失败
- 紧急支持升级
- 由无代码工作流检测到的服务故障
- 需要立刻回拨的线索提交

如果你的重点是突破专注模式，也可以继续看：[如何让关键告警突破 iOS 专注模式](/en/blog/how-to-bypass-ios-focus-mode-for-critical-alerts)。

## 特别适合和 Echobell 搭配的 Zapier 工作流

### 1. 高意向线索通知

来自 Typeform、Tally、HubSpot 或自定义表单的提交，适合直接推送到销售或创始人的手机。

### 2. 支付与结账告警

来自 Stripe、Shopify 或内部计费流程的失败事件，很适合被升级成更强通知。

### 3. 支持升级

来自 Help Scout、Zendesk、Intercom 或 Gmail 标签规则的事件，可以直接送到需要处理的人手机上。

### 4. 内部运维自动化

Zapier 监测 SaaS、表格或无代码系统，再把结果发给 Echobell，能有效补强内部运营告警。

## 最佳实践

### 一个工作流一个频道

不要把所有 Zap 都塞进一个频道。至少按业务重要性拆开。

例如：

- `紧急线索`
- `收入故障`
- `支持升级`
- `内部运维`

### 尽量带上返回源系统的链接

如果 payload 里有 `externalLink`，Echobell 能让接收者更快回到原始工单、线索或记录页面。

### 只把真正昂贵的事件升级成电话

如果每个 Zap 都打电话，团队很快就会失去对这条信号的信任。电话提醒应该只留给最少数、最关键的事件。

## 最后建议

Zapier 已经很擅长把数据从应用里拿出来，真正的问题是：这些事件到你手机上时，是否有正确的紧急度。

如果你有哪怕一个自动化流程“太重要，不适合只发邮件或普通推送”，那就把它接进 Echobell：

- [iPhone Webhook 通知](/en/features/webhooks)
- [关键事件电话告警](/en/features/call-notifications)
- [服务器宕机电话告警](/en/server-down-phone-call-alerts)

很多时候，这个改动带来的价值，比你再多堆一层自动化逻辑还大。
