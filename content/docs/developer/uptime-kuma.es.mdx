[Uptime Kuma](https://github.com/louislam/uptime-kuma) es una herramienta de monitoreo autohospedada fácil de usar. Puede recibir notificaciones a través de Echobell cuando sus servicios dejen de funcionar.

<Steps>
  <Step>
    ### Crear un canal
    Cree un nuevo canal en Echobell y copie la **URL del Webhook**.
  </Step>
  <Step>
    ### Configurar notificación en Uptime Kuma
    1. Vaya a **Ajustes** > **Notificaciones**.
    2. Haga clic en **Configurar Notificación**.
    3. Seleccione **Webhook** como Tipo de Notificación.
    4. Establezca el **Nombre Amigable** como "Echobell".
    5. Pegue su URL de Webhook de Echobell en **Post URL**.
    6. Establezca el **Método** en `POST`.
    7. Establezca el **Tipo de Cuerpo** en `JSON`.
    8. Habilite **Cuerpo Personalizado**.
  </Step>
  <Step>
    ### Configurar Payload
    Ingrese el siguiente JSON en el campo **Cuerpo Personalizado**:

    ```json
    {
      "title": "Monitor {{name}} is {{status}}",
      "body": "{{msg}}\nURL: {{hostnameOrURL}}",
      "status": "{{status}}",
      "monitor": "{{name}}"
    }
    ```

    Puede personalizar la estructura JSON según sea necesario. Echobell utilizará estas variables en sus plantillas de notificación.

  </Step>
  <Step>
    ### Probar
    Haga clic en **Probar** para enviar una notificación de prueba a Echobell.
  </Step>
</Steps>
