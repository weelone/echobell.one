# メールトリガー

メールトリガーは、プログラミングの知識を必要とせずに Echobell で通知を送信する簡単な方法を提供します。各チャンネルは専用のメールアドレスを持つことができ、このアドレスにメールが送信されると、自動的にそのチャンネルのすべての購読者に通知がトリガーされます。

## メールトリガーの仕組み

チャンネルにメールトリガーを設定すると：

1. チャンネルに固有のメールアドレスが割り当てられます
2. このアドレスに送信されたメールはチャンネルをトリガーします
3. メールの内容は自動的に変数として解析されます
4. これらの変数は通知テンプレートで使用できます

## 利用可能なメール変数

メールがチャンネルをトリガーすると、Echobell は以下の変数を自動的に抽出し、通知テンプレートで使用できるようにします：

- `from`: 送信者のメールアドレス
- `to`: 受信者のメールアドレス（チャンネルのメールアドレス）
- `subject`: メールの件名
- `text`: メールのプレーンテキスト内容
- `html`: メールの HTML 内容（利用可能な場合）

## メール変数のテンプレートでの使用

これらの変数は、`{{変数名}}`構文を使用してテンプレートで使用できます。例：

```
タイトル：{{from}}からの新規メール
本文：件名：{{subject}}

{{text}}
```

## ユースケース

メールトリガーは以下のような用途に特に便利です：

- 重要なメールを通知として転送
- プログラミングなしで通知フローを作成
- メールはサポートしているが webhook をサポートしていないレガシーシステムとの統合
- メールベースの監視システムからのアラートの設定
- ニュースレターをモバイル通知に変換

<Callout type="info">
  メール送信者が通知をトリガーする前にアドレスを確認する必要がある場合、確認メールの記録にリンクがあります。リンクを開いてメールの内容を確認し、メールアドレスを検証してください。
</Callout>

## メール統合のセットアップ

1. チャンネル設定に移動
2. メールトリガーを有効化
3. 生成されたメールアドレスをコピー
4. メールフォワーディングを設定するか、システムにメールアドレスを統合

## プライバシーとデータストレージ

Echobell のすべての機能と同様に：

- チャンネル設定データのみがサーバーに保存されます
- 通知とメールトリガーの内容はデバイスにローカルに保存されます
- メールデータはあなたの管理下を離れることはありません

## ベストプラクティス

1. 異なるチャンネルには明確で具体的なメールアドレスを使用してください
2. メール内容に合った適切な通知テンプレートを設定してください
3. 緊急なメールには時間重視または通話タイプの通知の使用を検討してください
4. 本番環境で使用する前に、テストメールでメールトリガーの設定をテストしてください

## トラブルシューティング

メールトリガーが期待通りに動作しない場合は、以下を確認してください：

1. メールアドレスが正しくコピーされていること
2. メール転送のルールが正しく設定されていること
3. チャンネルがアクティブであること
4. テンプレートがメール変数を正しく使用していること
5. 通知を受信するアクティブな購読者がいること

## 次のステップ

- より高度なテンプレートオプションを[通知ガイド](/ja/docs/notification)で探索
- メールトリガーと[webhook](/ja/docs/webhook)を組み合わせて柔軟な通知フローを作成
- チャンネルをチームメンバーと共有して効率的な通知管理を実現
