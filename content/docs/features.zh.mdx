## 核心功能

### 通知频道

Echobell 的核心是频道 - 您可以为不同目的创建的可定制通知管道。每个频道具有：

- 便于识别的独特名称和颜色
- 可定制的通知模板
- 灵活的触发选项（webhook 或邮件）
- 可共享的订阅链接

### 触发方式

#### Webhook 触发器

使用 webhook URL 与几乎任何系统集成。当这些 URL 被调用时，您的频道会向所有订阅者发送通知。特别适用于：

- 服务器监控警报
- CI/CD 流程通知
- IoT 设备状态更新
- 电商订单通知

您可以通过查询参数或 JSON 正文提供变量：

```http
GET https://hook.echobell.one/t/xxx?server_name=Production&cpu_usage=95

# 或者

POST https://hook.echobell.one/t/xxx
Content-Type: application/json

{
  "server_name": "Production",
  "cpu_usage": 95
}
```

特殊变量：

- `externalLink`：包含时，在通知记录中创建可点击的链接

#### 邮件触发器

每个频道可以有专用的电子邮件地址。当向此地址发送邮件时，频道就会被触发。这样可以：

- 将重要邮件转发为通知
- 无需编码创建通知流程
- 与支持邮件但不支持 webhook 的旧系统集成

可用的邮件变量：

- `from`：发件人邮箱地址
- `subject`：邮件主题行
- `text`：纯文本内容
- `html`：HTML 内容

### 通知类型

订阅者可以根据紧急程度选择三种通知类型：

1. **普通**

   - 用于日常提醒的标准应用通知
   - 遵循您设备的常规通知设置

2. **时效性**

   - 高优先级通知
   - 可以突破某些专注模式
   - 用于不应错过的重要提醒

3. **来电**
   - 针对关键情况的类似来电提醒
   - 需要立即注意
   - 在您的设备上显示为来电

### 模板系统

使用带变量的可定制模板使您的通知动态化：

- **标题模板**：决定通知标题中显示的内容
- **正文模板**：控制通知的主要内容

模板示例：

```
标题：服务器 {{server_name}} 已宕机！
正文：CPU使用率达到 {{cpu_usage}}%
```

变量用双大括号 `{{变量名}}` 括起来，由触发数据填充。

### 隐私功能

Echobell 的核心建立在隐私保护之上：

- 服务器只存储用户账户、频道和订阅数据
- 通知内容和历史记录完全保存在您的设备上
- 您的敏感提醒数据永远不会离开您的控制范围

### 应用设置

自定义通知行为：

- **重复语音内容**：对于来电通知，启用重复通知内容
- **重试失败的来电**：自动重试因专注模式失败的来电通知

## 使用场景

### 开发人员

- 监控服务器健康状况并接收即时警报
- 获取构建完成或测试失败的通知
- 接收关键生产错误的警报

### 团队

- 为不同的项目或系统创建频道
- 让团队成员按照首选的通知紧急程度订阅
- 确保重要提醒永不错过

### 个人使用

- 在网站宕机时获取警报
- 接收重要邮件的通知
- 为任何重要事项创建自定义提醒
