# Echobellを始めよう

私たちは皆、同じような経験をしています - サーバーが何時間もダウンしていたことに気づいたときの嫌な感じ、コーヒーを取りに行っている間にCI/CDパイプラインが失敗していたときの感じ。さまざまなツールからの終わりのない通知の流れが、アラートに対する私たちの感覚を麻痺させ、本当に重要なものを見逃すことになってしまいました。

まさにそれが、私たちがEchobellを作った理由です。偽のアラームで起こされたり、重要なアラートを完全に見逃したりした何年もの経験の後、インスタント通知を扱うより良い方法があるはずだと知っていました。

## Echobellが違う理由は？

Echobellは、開発者が本当に必要としているものに焦点を当てて、アラートと通知の管理方法を変革します：

- **スマートなwebhook通知** - 重要なときだけトリガーされる
- **メール-to-通知ブリッジ** - レガシーシステム用
- **カスタマイズ可能なアラートチャンネル** - 完全に制御できる
- **プライバシーファーストデザイン** - 通知データはデバイスに残る

個人的な通知執事だと考えてください - いつあなたを邪魔し、いつ静かにしているべきかを知るほど賢いのです。

## Echobellの中核概念を理解する

あなたの日々のワークフローに意味をなすように、Echobellがどのように機能するかを分解しましょう：

### チャンネル - あなたの通知コマンドセンター

**チャンネル**をシステムの異なる側面のための専用通知パイプラインと考えてください：

- 各チャンネルには独特の**名前と色**があります（緊急時に視覚的認識が貴重な秒数を節約するため）
- **通知テンプレート**は、正確にどの情報があなたの電話に送信されるかを定義します
- チャンネルはwebhookまたは良き古きメールでトリガーできます
- チームとチャンネルを共有したり、他の人が作成したチャンネルを購読したりできます

*実際の例：「本番アラート」チャンネル（赤）を重要なシステム問題用に、「デプロイ更新」チャンネル（緑）を成功したデプロイ用に持つかもしれません。*

### トリガー - あなたのシステムがあなたと話す方法

**トリガー**は、あなたのシステムがチャンネルを起動する方法です：

- **Webhook URL**: 監視ツール、CI/CDパイプライン、または任意のシステムがこのURLを呼び出して、インスタント通知を送信できます
- **メールアドレス**: 重要なメールを転送したり、メールしか送信できないシステムと統合したりできます

*なぜ両方？時にはREST APIを話すモダンなマイクロサービスを扱い、時には2015年のSMTPしか知らないレガシーシステムを扱います。*

### 通知タイプ - 緊急度を現実に合わせる

チャンネルを購読するとき、どれほど緊急に中断されたいかを選択します：

- **通常**: ルーチン更新のための標準プッシュ通知
- **時間重視**: 重要だが重要でないアラートのために「お邪魔しません」を突破
- **通話**: 「午前3時に起こして」レベルの緊急事態のための電話アラート

*プロのヒント: 通話通知は真の緊急事態のために予約してください。偽のアラーム通話ほど信頼を破るものはありません。*

## はじめに - Echobellでの最初の5分間

最初の通知チャンネルを設定しましょう。Prometheusアラートを設定するより簡単だと約束します！

### 1. アカウントをダウンロードして設定

1. **[App Store](https://apps.apple.com/app/id6743597198)からEchobellをダウンロード**（はい、無料です）
2. **メールまたはApple ID**でサインイン - より快適に感じる方で

![アカウント設定](/images/docs/screenshot-sign-in.en.webp)

*クイックノート: アカウント管理のためだけにメールを使用します。スパムなし、明示的に要求されない限りニュースレターもなし。*

### 2. 最初の通知チャンネルを作成

ここで魔法が起こります：

1. **右下のボタンをタップ**して**チャンネル**ビューに移動
2. **+ボタンをタップ** - 最初のチャンネルを作成する時間です
3. **チャンネルに意味のある名前を付ける**（例：「本番サーバーアラート」や「CI/CDパイプライン」）
4. **意味のある色を選択**（重要な場合は赤、成功の場合は緑など）
5. **通知テンプレートを設定**（下で詳しく説明します）
6. **このチャンネルに自動購読するかどうかを選択**
7. **通知緊急度レベルを選択**（通常、時間重視、または通話）
8. **保存**して祝います - 最初のスマート通知チャンネルを作成しました！

![チャンネル作成](/images/docs/screenshot-create-channel.en.webp)

*ヒント: 最初にテストチャンネルから始めてください。「テストアラート」のような名前を付けて、チーム全体に誤って重要なアラートを送信せずに実験できるようにします。*

### 3. トリガーでシステムを接続

ここでEchobellが本当に強力になります。**新しく作成されたチャンネルをタップ**して設定ページにアクセス - これがあなたの通知コントロールセンターです。

![チャンネル設定](/images/docs/screenshot-channel-settings.en.webp)

#### オプションA: Webhookトリガー（モダンな方法）

モダンなシステム、監視ツール、HTTP要求を行えるものに最適：

1. **webhookボタンをタップ**して一意のwebhook URLをコピー
2. **このURLをシステムに統合**：
   - Grafanaアラート
   - GitHub Actionsワークフロー
   - Kubernetes監視
   - カスタムスクリプトと監視ツール
   - HTTP呼び出しを行える任意のシステム

通知をトリガーするのは簡単です：

```bash
# アラートデータを含むシンプルなGET要求
curl "https://hook.echobell.one/t/your-unique-id?server=web-01&status=DOWN&cpu=95%"

# または複雑なデータ用のJSONを含むPOST
curl -X POST https://hook.echobell.one/t/your-unique-id \
  -H "Content-Type: application/json" \
  -d '{
    "server": "web-01",
    "status": "重要", 
    "message": "CPU使用率が95%に上昇",
    "dashboard_url": "https://grafana.company.com/dashboard/123"
  }'
```

**プロのヒント:** webhook呼び出しに`externalLink`変数を含めてください。通知内でクリック可能なリンクになり、ダッシュボードやログに直接ジャンプするのに最適です。

#### オプションB: メールトリガー（汎用フォールバック）

時にはREST APIより古いシステムで作業する必要があります：

1. **メールボタンをタップ**してチャンネルの一意のメールアドレスを表示
2. **重要なメールをこのアドレスに転送**するか、レガシーシステムを設定してここにアラートを送信
3. メールが届くと、これらの変数で自動的に通知をトリガーします：
   - `from` - メールを送信した人
   - `subject` - メールの件名行
   - `text` - プレーンテキストメールコンテンツ
   - `html` - フォーマットされたメールコンテンツ

*これは古いシステムとの統合、重要なメールの転送、またはコードを書かずに通知ワークフローを作成するのに非常に有用です。*

### 4. 完璧な通知メッセージを作成

ここで生データを実行可能なアラートに変換します。良い通知テンプレートは、有用なアラートと通知スパムの違いです。

**テンプレートの基本:**
1. **チャンネル設定の通知テンプレートに移動**
2. **タイトルテンプレート** - 簡潔だが情報的に保つ（通知プレビューに表示されます）
3. **本文テンプレート** - アクションを取るために必要な詳細を含める
4. **`{{二重中括弧}}`で囲まれた変数を使用**してテンプレートを動的にする

**機能する実世界の例:**

```
タイトル: 🚨 {{server}}に問題があります
本文: ステータス: {{status}} | CPU: {{cpu}}% | メモリ: {{memory}}% | 確認: {{dashboard_url}}
```

```  
タイトル: ✅ {{environment}}へのデプロイ完了
本文: {{app_name}} v{{version}}が正常にデプロイされました。期間: {{deploy_time}}
```

```
タイトル: 💸 新しい注文 #{{order_id}} 
本文: 顧客: {{customer_name}} | 金額: ${{amount}} | アイテム: {{item_count}}
```

**テンプレートプロのヒント:**
- 迅速な視覚的認識のために絵文字またはステータスインジケーターで始める
- 実行可能な情報を含める（URL、サーバー名、エラーコード）
- より良いモバイル表示のためにタイトルを50文字以下に保つ
- 本番稼働前に実際のデータでテンプレートをテストする

### 5. チャンネルを共有し、通知ネットワークを構築

Echobellの最も強力な機能の1つは協調通知です：

**他の人のチャンネルを購読するには:**
1. **チャンネルビューで購読をタップ**
2. **共有された購読リンクを貼り付け**
3. **通知緊急度レベルを選択**（チャンネル所有者ではなく、あなたが制御します）
4. **確認** - 彼らの通知を受信するようになります

**チャンネルを共有するには:**
1. **チャンネル設定を開く**
2. **購読リンクをコピー**
3. **Slack、メール、またはチームのコミュニケーション方法でチームメンバーと共有**

*これはチームアラート、デプロイ通知、または部門間での監視データ共有に素晴らしいです。*

## 通知エクスペリエンスの微調整

### 本当に重要なアプリ設定

アプリ設定を通じてこれらにアクセスして、Echobellエクスペリエンスをカスタマイズ：

- **音声コンテンツを繰り返す**: 通話通知を受けるとき、これはすべての詳細をキャッチできるようにメッセージを繰り返します（午前3時に起こされたときに有用）
- **失敗した通話を再試行**: 「お邪魔しません」または他のフォーカスモードで失敗した通話通知を自動的に再試行

*小さな設定、信頼性への大きな影響。*

## あなたのプライバシーが最優先

私たちも開発者なので、システムデータがどれほど機密性が高いかを知っているため、プライバシーファーストの考え方でEchobellを構築しました：

- **サーバー側データ**: あなたのアカウント、チャンネル定義、購読リストのみ
- **デバイス専用データ**: すべての通知コンテンツと履歴は完全にローカルに残る
- **あなたの制御**: アラートデータは決してあなたの制御を離れることも、私たちによって分析されることもありません

*翻訳: 「本番アラート」というチャンネルを作成したことは見えますが、「データベースCPUが95%」のような実際のアラートコンテンツは決して見ません。*

## 次は何？アラートゲームをレベルアップ

最初のチャンネルが動作するようになったので、通知設定を進化させる方法をいくつか紹介します：

**即座の次のステップ:**
- 異なる重要度レベル（情報、警告、重要）のチャンネルを作成
- 既存の監視ツールとのwebhook統合を設定
- チームメンバーと重要なチャンネルを共有

**高度な戦略:**
- アラートデータに基づいてメッセージをカスタマイズする条件付きテンプレートを使用
- 異なる通知タイプでエスカレーションワークフローを作成
- 複雑なシステムのためのチャンネル階層を構築

**統合のアイデア:**
- デプロイ通知のためのCI/CDパイプラインを接続
- レガシーメールアラートをモダンプッシュ通知にブリッジ
- 電話に直接話しかける監視ダッシュボードを設定

## ヘルプが必要？私たちがいます

すべての開発者のセットアップは独特で、人々がEchobellから最大限を得るのを支援するのが大好きです：

- **[機能ドキュメンテーション](/docs/features)で詳細ガイドを探索**
- **ブログ投稿の統合チュートリアルをチェック**
- **[サポートセンター](/docs/support)で直接ヘルプを取得**
- **echobell@weelone.comに直接メール**（実際にメールを読んで返信します！）

覚えておいてください：素晴らしい通知システムは一日では構築されませんが、投資する価値は間違いなくあります。あなたの未来の午前3時の自分は、これを適切に設定してくれたことに感謝するでしょう。

### 1. アカウントの設定

1. [App Store](https://apps.apple.com/app/id6743597198)から Echobell をダウンロード
2. アプリを開き、メールまたは Apple でサインイン

![アカウント設定](/images/docs/screenshot-sign-in.en.webp)

### 2. 最初のチャンネルを作成

1. ホーム画面で「チャンネル作成」をタップ
2. チャンネルの名前を入力（例：「サーバー監視」）
3. チャンネルに容易に識別できる色を選択
4. チャンネルが作成され、設定画面が表示されます

![チャンネル作成](/images/docs/screenshot-create-channel.en.webp)

### 3. トリガーの設定

1. チャンネル設定画面で「トリガー」へ移動
2. **Webhook** または **メール** トリガーを選択:
   - **Webhook**: URL をコピーしてシステムに統合
   - **メール**: メールアドレスをコピーして転送ルールを設定

### 4. 通知テンプレートのカスタマイズ

トリガーデータから入力される変数を使用するテンプレート：

1. チャンネル設定で、**通知テンプレート**へ移動
2. タイトルテンプレートには：`{{server}} - {{status}}`のように設定
3. 本文テンプレートには：`{{message}} {{timestamp}}に検出`のように設定
4. 変数は二重中括弧で囲みます：`{{変数名}}`
5. テンプレートをサンプルデータでテスト

### 5. チャンネルの購読

1. チャンネルの購読リンクを他のユーザーと共有できます
2. または他のユーザーが共有した購読リンクを使用してチャンネルを購読できます
3. チャンネルビューで**購読**をタップ
4. 購読リンクを入力
5. 希望する**通知タイプ**を選択：
   - 標準的なアラートには通常
   - 重要なアラートには時間重視
   - 重要な状況には通話
6. 購読を確認

## アプリの設定

以下の設定で Echobell の通知処理方法をカスタマイズできます：

- **音声内容の繰り返し**: 通話通知を使用する際、通知内容を繰り返します
- **失敗した通話の再試行**: フォーカスモードなどの理由で失敗した通話通知を自動的に再試行します

## プライバシーとデータストレージ

Echobell はプライバシーを優先します：

- サーバーには、ユーザーアカウント、チャンネル、購読データのみを保存
- 通知のコンテンツと履歴はあなたのデバイスにローカルに保存
- アラートデータはあなたの管理下を離れることはありません

## 次のステップ

最初のチャンネルを設定したら：

- 高度なテンプレートオプションを探索
- チームメンバーとチャンネルを共有
- 通知スケジュールと静寂時間を設定

各機能の詳細については、[機能ガイド](/ja/docs/features)セクションをご覧ください。

ヘルプが必要ですか？[サポートセンター](/ja/docs/support)にアクセスするか、echobell@weelone.comまでメールでお問い合わせください。
