# Echobell をはじめよう

Echobell へようこそ。重要な通知を常に把握できるようにサポートするインスタントアラートアプリです。このガイドでは、Echobell の効果的な設定と使用方法について説明します。

## Echobell とは？

Echobell は以下のような強力な通知管理ツールです：

- webhook やメールをトリガーとして処理
- 購読者にカスタマイズされた通知を送信
- 通知チャンネルの作成と共有が可能
- 通知データをデバイス上でプライベートに保持

![Echobellの概要](/images/docs/screenshot-overview.en.webp)

## 主要概念

利用を開始する前に、いくつかの重要な概念を理解しましょう：

### チャンネル

**チャンネル**は、通知を整理・配信するための Echobell の基本単位です：

- 各チャンネルには、容易な識別のための固有の**名前**と**色**があります
- チャンネルには、送信される情報を決定する**通知テンプレート**が含まれます
- チャンネルは webhook またはメールでトリガーされます
- 自分でチャンネルを作成したり、他のチャンネルを購読したりできます

### トリガー

**トリガー**は、チャンネルをアクティブにして通知を送信するものです：

- **Webhook**: 呼び出されるとチャンネルをアクティブにする URL エンドポイント
- **メール**: メッセージを受信するとチャンネルをアクティブにする、チャンネルにリンクされたメールアドレス

### 通知タイプ

チャンネルを購読する際、異なる通知スタイルから選択できます：

- **通常**: 標準的なアプリ通知
- **時間重視**: フォーカスモードを突破できる高優先度通知
- **通話**: 最も緊急な事項のための電話のようなアラート

## はじめ方

### 1. アカウントの設定

1. [App Store](https://apps.apple.com/app/id6743597198)から Echobell をダウンロード
2. アプリを開き、メールまたは Apple でサインイン

![アカウント設定](/images/docs/screenshot-sign-in.en.webp)

### 2. 最初のチャンネルを作成

1. ホーム画面で「チャンネル作成」をタップ
2. チャンネルの名前を入力（例：「サーバー監視」）
3. チャンネルに容易に識別できる色を選択
4. チャンネルが作成され、設定画面が表示されます

![チャンネル作成](/images/docs/screenshot-create-channel.en.webp)

### 3. トリガーの設定

1. チャンネル設定画面で「トリガー」へ移動
2. **Webhook** または **メール** トリガーを選択:
   - **Webhook**: URL をコピーしてシステムに統合
   - **メール**: メールアドレスをコピーして転送ルールを設定

### 4. 通知テンプレートのカスタマイズ

トリガーデータから入力される変数を使用するテンプレート：

1. チャンネル設定で、**通知テンプレート**へ移動
2. タイトルテンプレートには：`{{server}} - {{status}}`のように設定
3. 本文テンプレートには：`{{message}} {{timestamp}}に検出`のように設定
4. 変数は二重中括弧で囲みます：`{{変数名}}`
5. テンプレートをサンプルデータでテスト

### 5. チャンネルの購読

1. チャンネルの購読リンクを他のユーザーと共有できます
2. または他のユーザーが共有した購読リンクを使用してチャンネルを購読できます
3. チャンネルビューで**購読**をタップ
4. 購読リンクを入力
5. 希望する**通知タイプ**を選択：
   - 標準的なアラートには通常
   - 重要なアラートには時間重視
   - 重要な状況には通話
6. 購読を確認

## アプリの設定

以下の設定で Echobell の通知処理方法をカスタマイズできます：

- **音声内容の繰り返し**: 通話通知を使用する際、通知内容を繰り返します
- **失敗した通話の再試行**: フォーカスモードなどの理由で失敗した通話通知を自動的に再試行します

## プライバシーとデータストレージ

Echobell はプライバシーを優先します：

- サーバーには、ユーザーアカウント、チャンネル、購読データのみを保存
- 通知のコンテンツと履歴はあなたのデバイスにローカルに保存
- アラートデータはあなたの管理下を離れることはありません
