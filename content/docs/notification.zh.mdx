# Echobell 中的通知

通知是 Echobell 功能的核心。本指南解释了通知的工作原理、可用的不同类型以及如何根据您的需求自定义它们。

## 通知类型

订阅频道时，您可以根据紧急程度选择以下三种通知类型：

- **普通**：用于常规提醒的标准应用通知。这些通知遵循您设备的正常通知设置。
- **时效性**：可以突破某些专注模式的高优先级通知。适用于不应错过的重要提醒。
- **来电**：用于需要立即关注的关键情况的类似来电的提醒。这些通知在您的设备上显示为来电。

## 通知模板

每个频道都使用可自定义的模板来设置通知内容：

- **标题模板**：决定通知标题中显示的内容
- **正文模板**：控制通知的主要内容

模板支持使用 `{{variableName}}` 语法的动态变量。例如：

```
标题：{{server}} 状态提醒
正文：在 {{timestamp}} 检测到 {{message}}
```

## 通知中的变量

### Webhook 变量

通过 webhook 触发通知时，您可以通过以下方式提供变量：

- 查询参数：`https://hook.echobell.one/t/xxx?server=production&status=critical`
- JSON 正文：

```json
{
  "server": "production",
  "status": "critical"
}
```

特殊变量：

- `externalLink`：包含时，在通知记录视图中创建可点击的链接

### 电子邮件变量

通过电子邮件触发时，以下变量自动可用：

- `from`：发件人电子邮件地址
- `to`：收件人电子邮件地址
- `subject`：电子邮件主题行
- `text`：纯文本内容
- `html`：电子邮件的 HTML 内容

## 隐私和存储

Echobell 采用隐私优先的方式处理通知：

- 通知内容和历史记录仅存储在您的本地设备上
- 服务器仅存储必要数据（用户账户、频道配置和订阅）
- 您的通知数据始终在您的控制之下

## 应用设置

通过以下设置配置通知行为：

- **重复语音内容**：对于来电通知，启用重复通知内容
- **重试失败的来电**：自动重试因专注模式或其他原因失败的来电通知

## 重要说明

- 应用需要通知权限才能正常运行
- 频道触发时实时发送通知
- 您可以接收来自您创建的频道和您订阅的频道的通知
- 不同的通知类型可用于不同的紧急程度
