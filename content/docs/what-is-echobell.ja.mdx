# Echobell とは？

Echobell は、通知管理を簡素化することを目的に設計されたインスタントアラートアプリケーションです。webhook やメールを処理してカスタマイズされた通知を送信することで、重要なアラートを見逃すことがないようにします。

## 主要機能

- **即座のアラート**: チャンネルがトリガーされた際にリアルタイムで通知を受信
- **複数のトリガー方法**: webhook またはメールベースのトリガーを設定
- **カスタマイズ可能な通知**: 動的変数を持つテンプレートを設定
- **購読システム**: チャンネルを作成して他のユーザーが購読可能
- **プライバシー重視**: 必要最低限のデータのみをサーバーに保存し、通知履歴はデバイスに保存

## Echobell の仕組み

webhook が呼び出されるかメールを受信すると、Echobell は受信データを処理し、そのチャンネルのすべての購読者に通知を送信します。通知のコンテンツは、トリガーで提供される変数とテンプレートに基づいて動的に生成されます。

## 主要概念

### チャンネル

チャンネルは Echobell の中核となるエンティティです。各チャンネル：

- 容易な識別のため、固有の名前と色を持つ
- タイトルと本文のコンテンツのための通知テンプレートを含む
- webhook またはメール経由でトリガー可能
- 購読リンクを使用して他のユーザーと共有可能

### 通知タイプ

チャンネルを購読する際、3 つの通知タイプから選択できます：

- **通常**: 標準的なアプリ通知
- **時間重視**: 特定のフォーカスモードを突破できる通知
- **通話**: 重要な通知用の電話のようなアラート

### テンプレート変数

テンプレートで変数を使用して、通知を動的にします：

```
タイトル：サーバー{{server_name}}がダウン！
本文：CPU使用率が{{cpu_usage}}%に到達
```

適切な変数でトリガーされると、これらのテンプレートはパーソナライズされた通知を生成します。

### トリガー方法

Echobell は 2 つの主要なトリガー方法をサポートしています：

- **Webhook**: システムを統合するための HTTP エンドポイント
- **メール**: 通知をトリガーするための専用メールアドレス

## プライバシーとデータストレージ

Echobell はプライバシーを念頭に設計されています：

- ユーザーデータ、チャンネル設定、購読情報のみをサーバーに保存
- 通知履歴と記録は、ローカルデバイスにのみ保存されます

## はじめ方

Echobell を使い始めるには：

1. [App Store](https://apps.apple.com/app/id6743597198)から Echobell アプリをダウンロード
2. メールまたは Apple ID でサインイン
3. チャンネルを作成して設定をカスタマイズ
4. webhook またはメールトリガーを統合
5. 通知の受信を開始

<Callout type="warn">
  Echobellはすべての機能が通知ベースであるため、正常に動作するには通知権限が必要です。
</Callout>
